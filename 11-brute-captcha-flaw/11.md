## 漏洞点

登录页有 4 位数字图片验证码。

它有两个问题，叠加后让验证码形同虚设：

1. **验证码可复用**：在同一个 session 里，一次识别出的验证码提交错误密码后不会失效，可以一直复用去试整本密码字典。这才是本题真正的漏洞——验证码退化成摆设，无法阻止自动化爆破。
2. **验证码极易识别**：图片是 60×25 的纯灰底、固定字体、无噪点无干扰线，每个数字的像素都完全一致。这种验证码连神经网络 OCR 都不需要，用一组内置的数字位图做模板匹配就能 100% 识别。

只要注意一点：

> 获取验证码图片和提交登录表单必须使用同一个 session。

因为验证码和 `PHPSESSID` 绑定，如果换 session，验证码就对不上。

## 手工复刻步骤

### 1. 进入题目目录

```bash
cd /Users/zlux/Project/Active/gkdctf/11-brute-captcha-flaw
source /Users/zlux/Project/Active/gkdctf/.venv/bin/activate
```

### 2. 安装依赖

```bash
pip install requests pillow
```

不需要 `ddddocr`/`onnxruntime`。本题验证码字体固定、无噪点，`main.py` 内置了 0-9 共 10 个数字的位图模板，用模板匹配识别，零额外依赖。

### 3. 先测试识别验证码

验证码识别逻辑：转灰度 → 二值化 → 按列切出每个数字 → 归一化到 14×18 → 和内置模板比汉明距离取最近的数字。`main.py` 里的 `recognize_captcha()` 就是做这件事，可以直接拿来测：

```python
import random
import requests
from main import recognize_captcha

session = requests.Session()
img = session.get(
    "http://目标地址/yanzhengma.php",
    params={"nocache": str(random.random())},
)

print(recognize_captcha(img.content))
```

能稳定输出 4 位数字就说明识别可用。

### 4. 观察登录请求

登录请求接口是：

```text
/login.php
```

提交字段是：

```text
name
pwd
yzm
```

所以爆破时提交：

```python
data = {
    "name": "administrator",
    "pwd": password,
    "yzm": code,
}
```

### 5. 爆破密码

题目目录里有密码字典：

```text
password2.txt
```

每次尝试流程：

1. 用同一个 `session` 请求验证码图片。
2. 用模板匹配 `recognize_captcha()` 识别验证码。
3. 用同一个 `session` 提交用户名、密码、验证码。
4. 如果页面返回 flag，说明成功。
5. 如果验证码错误，就重新拉验证码再试当前密码。

直接运行 `python main.py`（或用 `GKD_URL` 指定目标地址）即可自动完成上述流程。
