## 题目背景

> Note  
> 取到校园站点，首页只有一句“Site Under Construction”。绝大多数新手到这会以为没东西直接跳过——真正的攻面在“开发过程留下的泄漏文件”里。
>
> 本题考察的正是渗透测试中最先要做的一步：目录 / 备份文件爆破。

## 漏洞本质

开发者将源码打包为 `[备份文件名占位]` 放在 web 根目录临时调试/传输，发布后忘删 → 整站源码直接对公网。备份包里还加了文档，明写“管理员入口路径”一直按带路。

| 泄露物 | 导致后果 |
|---|---|
| `[备份文件名占位]` | 整站源码 + 配置 + 密钥全失 |
| `README.txt`【包内】 | 隐藏后台路径裸美 |
| `[隐藏入口文件占位]` | 普通字典扫不到，却被文档出卖 |

> Flag 位置：隐藏后台页面 `/[敏感路径占位]` 直接渲染 `$GZCTF_FLAG`，无鉴权。

# 默认字典 + 备份扩展名

> Tip

关键参数：

- `-e zip,tar.gz,rar,bak,swp,sql` — 自动为字典条目追加备份扩展名

- `--exclude-sizes=277` — 当防火墙用固定大小 404 页面混淆时过滤

## 常见坑

| 问题 | 根因 | 解决 |
|---|---|---|
| dirsearch 扫不出 `[备份文件名占位]` | 默认字典没包含 `zip` 扩展 | 加 `-e zip,tar.gz,rar,bak` |
| 所有路径 200 | 服务器配置 “万物 200” | 用 `-fs <默认 size>` 过滤同长响应 |
| 扫描太快被拉黑 | 并发 / 速率超限 | 降到 `-t 5` + `--delay 0.3` |
| 下载下来的备份包为 0 字节 | 直接用浏览器访问被拦 / Content-Disposition 丢失 | 用 `curl -O` |
| zip 解压需密码 | 开发者自己加了密码 | 用 `7z` + `john --wordlist=rockyou.txt` |

## 目录 / 文件爆破通用套路

Important

1. 每个靶场第一步先看 `/robots.txt` — 有的站会把 `/admin/` `/api/internal/` 直接写进 `Disallow:`
2. 备份文件字典要带上“年份/项目名/bak/old” — `[备份文件名占位]` / `[备份文件名占位]` / `[备份文件名占位]` 这种高价值文件名
3. 不单看状态码，看响应长度 — 许多站把 404 写成 200 统一模板页
4. 注意 `.git/`、`.svn/`、`.hg/` — 一旦下着 `HEAD` / `entries`，用 `GitTools/Dumper` 直接 “拉下整个仓库”
5. zip 内的 README 和 `.git` 里的 commit log 等同于“开发者日记” — 路径 / 密码 / 内网 IP 都能挂在里面

## 关键修复

Important

修一：开发流程禁止将源码包/调试文件留在 web 根目录，有 `.gitignore`：

```gitignore
*.zip
*.bak
*.swp
*.tar.gz
/.env
/.git/
/.idea/
/.DS_Store
```

修二：Apache/Nginx 配置封死敏感扩展名：

```apache
<FilesMatch "\.(zip|tar\.gz|rar|bak|swp|sql|git|svn|DS_Store|env)$">
  Require all denied
</FilesMatch>
```

修三：隐藏后台不是防护，正确做法是背后加身份验证 + IP 白名单 + `.htaccess basic auth`。“隐藏路径” 只是 弱因隐私（security-by-obscurity），任一泄漏点泄露即穿友。

修四：项目上线后定期 `dirsearch` 自查 — 在 CI 里合并为定时任务，发现敏感文件自动报警。
