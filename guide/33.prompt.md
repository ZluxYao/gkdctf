## 漏洞本质

# 关键前置知识

**密码重置流程的"六大攻击面"（CWE-640 相关子类）：**

| 攻击面 | 本题命中 | 经典案例 |
| --- | --- | --- |
| 1. Token 交付通道错位（本题） | ✅ | 开发遗留 debug 信息、异步邮件兜底 |
| 2. Token 可预测 | ❌ | 时间戳 / md5(username) / 自增 id |
| 3. Token 永久有效 / 不失效 | 可能 | Dropbox 2013 重置 URL 永久有效 |
| 4. Token 复用攻击 | 可能 | 用过一次还能再用 |
| 5. Host header 注入 | 常见 | 改 `Host: attacker.com` 让邮件里的 URL 指向攻击者 |
| 6. 响应差异账号枚举 | 部分 | "user not found" vs "已发送" 区分账号是否存在 |

**Token 交付通道的 6 种错位模式：**

| 错位方式 | 表现 | 检查 |
| --- | --- | --- |
| 响应 body 返 token（本题） | `<a href="/reset?token=XXX">` | Burp 看 `/forgot` 响应 |
| 响应 header 返 token | `X-Debug-Reset-URL: ...` | 看所有 response headers |
| 设置 cookie 含 token | `Set-Cookie: reset_token=XXX` | 看 set-cookie |
| JSON API 调试字段 | `{"token":"XXX","__debug":{...}}` | 格式化 JSON 看全部字段 |
| Host header 被 reflect 进邮件 URL | `Host: evil.com` → 邮件 URL `https://evil.com/reset?token=XXX` | 改 Host 发 |
| 邮件模板变量注入 | `& / <script>` 注入到 URL | 邮件二次分享 |

**Token 生成强度的"三必一可"：**

- **必须**：密码学随机（`secrets.token_hex/urandom`，不是 `random.random()`）
- **必须**：熵 ≥ 128 bit（`token_hex(16)` = 128 bit）
- **必须**：绑定到用户 + IP（可选）+ 时间
- **可选**：HMAC 签名防篡改

**Token 生命周期的"四状态"：**

[代码块边界]text
签发(Issued) —— 发送(Delivered) —— 使用(Used) —— 错毁(Invalidated)
                                            ↑
                                            └─ 同时触发：有效期到 / 点击使用 / 用户修改密码 / 登录成功
[代码块边界]

**真实历史 "重置 token 失控" 大事件：**

- Twitter 2016 - 重置链接在邮件转发时被缓存
- Dropbox 2013 - 重置 URL 永久有效
- Uber 2018 - Host header 注入把 token 发到攻击者域

- 国内某电商 2022 - `/forgot` 接口把 6 位短信验证码返回在响应里（号称"便于前端对比"）

## 常见坑

| 问题 | 根因 | 解决 |
| --- | --- | --- |
| password required 报错 | 字段名不是 `password` 是 `new_password` | 打开 `/reset` 看 form 原文 |

| token 不同但长度相同 | 正常的随机 hex | 不是弱随机 |
| nobody 返 404 admin 返 200 | 响应差异可枚举账号 | 不影响本题但是额外泄露 |
| /forgot 接口有频控 | 短时间重复请求 429 | 本题没，真实环境常有 |

修二：Token 生命周期硬约束

[代码块边界]python
def validate_reset_token(user, token):
    rec = RESET_TOKENS.get(user)
    if not rec: return False
    saved_token, ts = rec
    if not hmac.compare_digest(saved_token, token): return False   # 抗时序攻击
    if time.time() - ts > 600: return False                       # 10 分钟过期
    return True

def use_reset_token(user):
    RESET_TOKENS.pop(user, None)                                  # 用后即销
[代码块边界]
