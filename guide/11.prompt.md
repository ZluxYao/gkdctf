## 原始题目 Markdown

```markdown
# Q11 验证码缺陷爆破 (11-brute-captcha-flaw)

## 题目背景

本题为 Q10 的升级版，在弱口令爆破基础上加一层图形验证码。

## 漏洞本质

核心考点为“验证码绕过，退化为纯密码爆破”。

常见缺陷：

- 校验成功后 `$_SESSION['captcha']` 未 `unset()`，一次 OCR 永久复用。
- 弱字符集（4 位纯数字 + 简单背景）导致 OCR 准确率较高。
- 答案可能回显在 cookie 或 hidden field。

## 逐步解题

1. 对验证码图片做 OCR 或从页面/响应中确认验证码来源。
2. 组合账号 `admin`、识别出的验证码和密码字典循环 POST 登录。
3. 密码字典可使用 `0000-9999`。
4. 命中后读取响应中的 flag。

## 教学提示

本题的关键不是复杂绕过，而是让学生理解验证码必须在服务端正确校验，并且要与会话和请求绑定。如果验证码可复用、可预测或只在前端校验，就不能阻止自动化爆破。

```
