# 漏洞本质

"黑名单 vs 白名单"的哲学命题：

| 维度 | 黑名单 | 白名单 |
|---|---|---|
| 覆盖 | 列出"禁止的东西"——列不完 | 列出"允许的东西"——列完即安全 |
| 假设 | 我知道所有攻击手段 | 我知道所有合法输入 |
| 出错模式 | 漏一个就破 | 卡死一个合法也能改 |

\

shell 里绕黑名单之所以容易，因为 `POSIX shell` 本身是一个"等价表达式"迷宫：

- 同一个"空格"可以用 `${IFS}` / `<>` 重定向 / `{,}` 花括号展开 / Tab / 换行表现
- 同一个 `cat` 可以用 `tac`、`head`、`more`、`nl`、`rev`、`od -c` 达到 "读文件出字符"
- 同一个 `/path` 可以用相对路径、`${PWD}`、`echo $0 | cut` 拼、`$(pwd)`

\

本题三关的反直觉副作用：

1. 禁 `/` → 看似钉了路径 → 实际 `flag` 就在 `CWD`，根本不需要路径
2. 禁 `cat` → 多达 8 种替身仅换个字母
3. 禁 `flag` 关键词 → 作者已把文件名改成 `s3cr3t_out.txt`，黑名单形同虚设

\

> Important

`stripos` 的假阳判定：

- `strpos('Cat', 'cat')` → `false`（严格区分大小写）
- `stripos('Cat', 'cat')` → `0`（不敏感）
- 本题用 `stripos`：大小写绕不动，但依然抵不过字符串替换

# 关键前置知识

"空格"的 7 种替身：

# 黑名单绕进阶 8 谱

> Important

1. **大小写：** 黑名单串用 `in_array` 而非 `stripos` → `CAT` / `CaT` 绕
2. **反斜杠：** `c\at`、`c""at`、`c''at` → bash 去掉空转义符
3. **变量拼：** `a=c;b=at;$a$b file` → 黑名单看源码字符，shell 看运行时
4. **Base64 中转：** `echo Y2F0IC9mbGFn | base64 -d | sh`
5. **通配符：** `/???/c??` → `/bin/cat`，完全无字母可禁
6. **printf 构字符串：** ``printf "\x63\x61\x74"`` → 运行时才实体化 cat
7. **环境变量间接：** `$PATH` 切片 → `${PATH:0:1}` = `/`，解除斜杠
8. **Here-doc / here-string：** `bash <<<'cat /flag'` → 整块代码由 bash 二次解析

# 关键修复

> Important

**修一（核心银弹）：白名单正则**

[代码块边界]php
// IP 地址严格格式
if (!filter_var($ip, FILTER_VALIDATE_IP)) exit('bad ip');

// 或 IP + 主机名白名单

- 黑名单是"反面清单"，永远列不完
- 白名单是"正面清单"，想绕须构造非期望内容——几乎无解

**修二：数组传参消除 shell**

[代码块边界]php

$out = stream_get_contents($pipes[1]);
proc_close($p);
[代码块边界]

- 根本不让 shell 参与拼接，所有元字符退化为字符串

**修三：`escapeshellarg` 做保底**

[代码块边界]php

- 自动单引号包裹 + 内部 `'` 转义
- 比黑名单可靠 N 个数量级

- 如果业务非要保留用户输入原样，删除非法字符后再拼：
