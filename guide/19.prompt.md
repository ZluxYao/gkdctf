# 漏洞本质

LFI 的两个重要利用角度：

| 读写 | Q18 | Q19 |
|---|---|---|
| 目标文件是什么 | 存着源码注释的 `index.php` | 攻击者自己写入的日志 |

日志条目实际落盘样：

[代码块边界]bash

`include` 执行时，PHP 解析器的行为：

1. 读到 `192.168.1.100 - - "GET / HTTP/1.1"` → 纯文本，原样输出
2. 读到 `<?php` → 切换到 PHP 代码解析模式
3. 执行 `system('id; cat /flag')` → 输出 `uid=…` 和 `[FLAG 占位]`
4. 遇到 `?>` → 切回文本模式，继续输出后续日志行

→ PHP 的“代码/数据边界模糊”在这里被滥用。任何文本文件只要包含 `<?php … ?>` 都会被当 PHP 跑。

# 日志污染 × LFI 进阶谱

> Important

1. **Apache/Nginx access.log（本题）**：UA / URI / Referer 三栏
2. **error.log**：不存在路径返 404，抽路径字段写入；但 PHP 错误也会翻混合流
3. **SSH auth.log**：用恶意用户名 `<?php …?>` SSH 登录（会被 `failed … user` 字样包裹）
4. **mail.log**：通过 `sendmail` / SMTP 发一封带 PHP 的邮件
5. **/proc/self/environ**：PHP < 5.3，代令 UA 是环境变量
6. **PHP session**：`session.upload_progress.enabled=On`，POST 的 PROGRESS_NAME 值被写入 `sess_xxx`
7. **文件上传临时文件**：POST 上传会产生 `/tmp/phpXXXXXX`；LFI 快速抽塌 include
8. **/tmp/nginx.log**：Nginx `error_log /tmp/nginx.log notice;`（小概率事件）
9. **data://**：若黑名单放开 data 伪协议→直接 `data://text/plain,<?php … ?>`
10. **phar://**：上传一个 `.jpg` 缝些 phar 格式→`phar://x.jpg/test` 后 `__destruct`
