## 漏洞本质

"归档里的路径是敌区数据" —— Zip Slip 的核心认识：

| 认知错位 | 开发者以为 | 实际 |
| --- | --- | --- |
| `zip` 条目名的来源 | 来自"合法的打包工具" | 完全由攻击者控制 |
| `os.path.join(base, name)` | "会把 `name` 夹进 `base`" | 当 `name` 含 `../` 时，`join` 按字符串拼，内核解析时 `../` 会跳出 `base` |
| 解压的边界 | "`zipfile` 库会帮我处理" | 只有调 `extract()` / `extractall()` 且新版 Python 才做过滤；自己 `open(target, 'wb')` 没人拦 |

# 关键前置知识

zip 条目名的 4 种“越界姿势”：

| 形式 | 示例 | 解压结果【从 `/app/extracted/<uuid>/` 出发】 |
| --- | --- | --- |
| 相对穿越 | `../../../../tmp/x.sh` | `/tmp/x.sh` |
| 绝对路径 | `/etc/cron.d/x` | 依赖实现，Python 新版会剥 `/`，老版直落 |
| Windows 反斜杠 | `..\..\x` | Windows 下生效；Linux 库基本吞掉 |
| 软链接 entry | zip 含符号链接 | 解压出软链 → 后续写链接目标穿越 |

Python `zipfile` 的 3 种解压姿势：

| 方法 | 是否防穿越 | 本题属 |
| --- | --- | --- |
| `ZipFile.extractall(path)` (Python 3.11+) | 已内置过滤 `..` | 否 |
| `ZipFile.extract(name, path)` | 会规范化名字，但历史版本有 bug | 否 |
| `手搓: open(os.path.join(base, name), 'wb')` | 零防护 | 本题 |

## 阶段 ②：找“写→读”的桥

我只能写 zip 里的文件。zip 解压后文件存哪：`/app/extracted/<uuid>/` 一但 `/view` 只能读 extracted 下，flag 在根目录下怎么办？

- 它跑一次 → stdout 写入 `_output.txt` → 落在 extracted 根

## 阶段 ③：如何“让运维脚本变成我的”？

## 常见坑

| 问题 | 根因 | 解决 |
| --- | --- | --- |
| `/view?name=../../../flag.txt` 返回 404 | `send_from_directory` 自带 `safe_join` | 放弃直读，走 Slip+trigger |
| 上传响应 500 | `os.makedirs(parent)` 在穿出目录写时可能 EACCES | trigger 落在容器可写目录(`/tmp/`)；避开 `/root`、`/etc` 等权限严的位置 |
| `_output.txt` 一直不出现 | trigger 文件名不对；或权限不对 | 多放几个候选名；脚本加 `#!/bin/bash` shebang |
| 新版 Python 上 `extractall` 自动过滤了 `..` | CVE-2007-4559 修好之后 | 题目手搓 `open(target,'wb')` 不受新版保护 |
| zip 是创建的但上传 400 "need a .zip file" | 扩展名不是 `.zip` | 文件名带 `.zip` 即可（本题 `lower().endswith('.zip')`） |
| payload 脚本里 `cat /flag.txt` 回显空 | 文件实际在 `/flag` 而非 `/flag.txt` | 脚本里加 `cat /flag /flag.txt 2>&1; ls -la /` 做盲探 |
| 第一次刷新没看到 `_output.txt` | 后台 2s 轮询还没跑到 | 多刷几次，或 sleep 3~5 再 `/list` |
| zip 里放了很多 `../` 条目被"文件名检查"拒 | 本题无此限制；若有可双重编码 entry 名 | 部分库对 entry 名做 UTF-8 normalize，可试 `..%2f` / 类似 |

修一【核心银弹】：解压前校验每一个 entry 名

[代码块边界]python
import os, zipfile
def safe_extract(zf, dest):
    dest = os.path.realpath(dest)
    for m in zf.infolist():
        target = os.path.realpath(os.path.join(dest, m.filename))
        if not target.startswith(dest + os.sep):
            raise Exception(f'blocked path traversal: {m.filename}')
    zf.extractall(dest)
[代码块边界]

- `realpath` 解决 `..` 和软链
- 前缀加 `os.sep` 防 `/tmp/extract2` 蹭 `/tmp/extract`

修二：拒绝绝对路径 + 拒绝 `..` 段 + 拒绝软链 entry

[代码块边界]python
for m in zf.infolist():
    if m.is_symlink():                  # Python 3.12+ 有 is_symlink
        raise Exception('symlink in archive')

    parts = m.filename.replace('\\', '/').split('/')
    if '' in parts[:-1] or '..' in parts:  # 绝对路径或越界
        raise Exception('unsafe path')

    if os.path.isabs(m.filename):
        raise Exception('absolute path')
