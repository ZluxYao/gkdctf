# 02 - 登录处 SQL 注入（认证绕过）

## 漏洞点

后台登录 `POST /` 把 `username` 直接拼进 SQL，密码校验可被注入短路。
测试账号 `guest / guest123`。

## 利用思路

手动注入，用户名填：

```
admin' or '1'='1
```

密码随便填。服务端 `302` 跳转到 `/admin.php` 并下发 `PHPSESSID`，
`admin.php` 直接显示 flag。

> 这题目录名是布尔盲注，但实际能直接绕过登录拿 flag，不用逐字符盲注。
> 真要盲注提数据，TRUE/FALSE 标志位是 `登录成功` / `登录失败`。
