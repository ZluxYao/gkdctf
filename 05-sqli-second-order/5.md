# 05 - 二次注入（Second-Order SQLi）

## 漏洞点

会员系统有注册 / 登录 / 改密三个接口：

- `register.php` 注册时用户名被**转义存库**（看起来安全）。
- `change.php` 改密时却直接把**库里取出的用户名**拼进 SQL，没再转义：

```sql
UPDATE users SET password='<newpass>' WHERE username='<从库里读出的用户名>'
```

注入点不在写入处而在「二次取用」处，所以叫二次注入。

## 利用思路

注册一个用户名本身就是 payload 的账号，登录后触发改密：

1. **注册**恶意用户名：
   ```
   pwn_xxx' OR username='admin'/*
   ```
   （`pwn_xxx` 随机前缀避免重名；`/*` 注释掉结尾单引号）
2. **登录**该用户，让会话绑定这个用户名。
3. **改密** `change.php` 时，存储的用户名被拼进 UPDATE，等价于：
   ```sql
   UPDATE users SET password='NewPass123456'
   WHERE username='pwn_xxx' OR username='admin'/*'
   ```
   `OR username='admin'` 让 admin 的密码也被改掉。
4. 用新密码**登录 admin**。
5. 访问 `admin.php` 读 flag。
